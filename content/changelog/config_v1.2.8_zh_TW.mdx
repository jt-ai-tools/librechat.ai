---
date: 2025-06-13
title: ⚙️ 配置版本 v1.2.8
version: "1.2.8"
---

- 為 OCR 配置新增了 `azure_mistral_ocr` 策略選項。
  - 啟用部署在 Azure AI Foundry 上的 Mistral OCR 模型。
  - 支援具備自定義終端的 Azure 特定部署配置。
  - 需要在 `mistralModel` 欄位指定 Azure 部署名稱。
  - 相容於 Azure AI Foundry 上提供的 Mistral OCR 2503 模型。
  - 詳情請參閱 [OCR 配置](/docs/configuration/librechat_yaml/object_structure/ocr_zh_TW)。

- 為 OCR 配置新增了 `vertexai_mistral_ocr` 策略選項。
  - 啟用部署在 Google Cloud Vertex AI 上的 Mistral OCR 模型。
  - 支援多種提供 Google Cloud 服務帳號憑證的方法：
    - 服務帳號 JSON 的檔案路徑。
    - 獲取服務帳號 JSON 的 URL。
    - Base64 編碼的服務帳號 JSON。
    - 原始 JSON 字串。
  - 自動從服務帳號憑證中提取專案 ID。
  - 使用 JWT 驗證安全存取 Vertex AI 終端。
  - 詳情請參閱 [OCR 配置](/docs/configuration/librechat_yaml/object_structure/ocr_zh_TW)。

- 強化了 MCP (模型上下文協定) 伺服器管理功能：
  - 在 MCP 伺服器物件結構中新增了 `serverInstructions` —— 控制在使用 MCP 伺服器工具時，是否將伺服器指令納入智慧體上下文。
  - 新增了使用者預留位置變數支援：
    - `{{LIBRECHAT_USER_ID}}` —— 替換為當前使用者的 ID，實現多使用者支援。
    - `{{LIBRECHAT_USER_*}}` —— 動態使用者欄位預留位置（將 `*` 替換為任何允許欄位的大寫形式）。
    - 支援於 `headers` 與 `url` 欄位。
  - 新增了按使用者憑證的 `customUserVars`：
    - 允許使用者為 MCP 伺服器安全地提供其自身的 API 金鑰與憑證。
    - 可透過工具選取處的設定圖標或 MCP 設定面板進行配置。
    - 確保在多使用者環境下身份驗證細節維持私密。
  - 在 `interface` 配置中新增了 `mcpServers`：
    - 集中管理 Model Context Protocol 伺服器相關的 UI 設定。
    - 包含 `placeholder` 選項，用以自定義未選取 MCP 時在伺服器下拉選單中顯示的文字。
  - 強化了伺服器管理，具備連線狀態追蹤與 OAuth 支援：
    - 新增了顯示伺服器狀態的動態狀態圖標（已連線、已斷開、需要 OAuth、錯誤、初始化中）。
    - 透過下拉選單與設定面板實作了支援 OAuth 流程的伺服器（重新）初始化。
    - 為 `customUserVars` 增加了已設定/未設定的狀態追蹤。
    - 統一了 MCPSelect 與 MCPPanel 的伺服器清單，並提供一致的狀態顯示。
  - 詳情請參閱 [MCP 伺服器物件結構](/docs/configuration/librechat_yaml/object_structure/mcp_servers_zh_TW)。

- 自定義終端標頭支援使用者預留位置變數：
  - 使用者現在可以在自定義終端標頭中使用相同的預留位置變數（`{{LIBRECHAT_USER_ID}}`, `{{LIBRECHAT_USER_EMAIL}}` 等）。
  - 詳情請參閱 [自定義終端物件結構 - 標頭](/docs/configuration/librechat_yaml/object_structure/custom_endpoint_zh_TW#headers)。

- 在介面配置中新增了 `temporaryChatRetention`：
  - 配置暫時對話的保留期（以小時為單位，最小 1，最大 8760）。
  - 若未指定，預設保留期為 720 小時（30 天）。
  - 可透過環境變數 `TEMP_CHAT_RETENTION_HOURS` 或在 `librechat.yaml` 中設定。
  - 僅適用於新建立的暫時對話；現有對話將保留原始的過期時間。
  - 詳情請參閱 [介面物件結構 - temporaryChatRetention](/docs/configuration/librechat_yaml/object_structure/interface_zh_TW#temporarychatretention)。

- 在 `fileConfig` 配置中新增了 `clientImageResize`：
  - 啟用了上傳前自動在用戶端縮放圖片的功能，以防止上傳失敗並優化效能。
  - 可配置最大尺寸 (`maxWidth`/`maxHeight`) 並保留寬高比。
  - 可調整 JPEG/WebP 壓縮品質 (0.1-1.0) 以優化檔案大小。
  - 支援選取輸出格式（JPEG 或 WebP）。
  - 有助於減少頻寬消耗並提升上傳速度。
  - 詳情請參閱 [檔案配置物件結構 - clientImageResize](/docs/configuration/librechat_yaml/object_structure/file_config_zh_TW#clientimageresize)。

- 強化了 `webSearch` 配置，提供全面的 Firecrawl 爬蟲選項：
  - 為 Firecrawl 爬蟲新增了詳細配置選項，包含 `formats`、`includeTags`、`excludeTags`、`headers`、`waitFor`、`timeout`、`maxAge`、`mobile`、`skipTlsVerification`、`parsePDF`、`removeBase64Images`、`blockAds`、`storeInCache`、`zeroDataRetention`、`onlyMainContent`、`location` 與 `changeTrackingOptions`。
  - 詳情請參閱 [網頁搜尋配置](/docs/configuration/librechat_yaml/object_structure/web_search_zh_TW)。

- 為所有終端新增了具備進階選項的標題產生配置：
  - 新增了 `titleMethod` 用以控制標題產生策略：
    - `"completion"`（新預設值） —— 使用標準補全 API 以實現更廣泛的 LLM 相容性。
    - `"structured"` 或 `"functions"`（舊版） —— 使用結構化輸出/函式呼叫（並非所有供應商都支援）。
  - 新增了 `titlePrompt` 用以自定義主標題產生的提示詞：
    - 必須包含用於對話內容的 `{convo}` 預留位置。
    - 允許完整控制 AI 產生標題的方式。
  - 新增了 `titlePromptTemplate` 用以自定義對話格式化方式：
    - 必須包含 `{input}` 與 `{output}` 預留位置。
    - 預設值：`"User: {input}
AI: {output}"`。
    - 控制對話在插入 `titlePrompt` 前的格式化方式。
  - 新增了 `titleEndpoint` 以使用替代終端進行標題產生：
    - 支援的數值：`openAI`、`azureOpenAI`、`google`、`anthropic`、`bedrock`。
    - 若為自定義終端，請使用配置中定義的精確自定義終端名稱。
    - 允許使用不同且可能更便宜的模型/終端來產生標題。
  - 所有的標題配置選項現已作為 [共用終端設定](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings_zh_TW) 使用。
  - 可使用 `all:` 終端定義進行全域配置。
  - 具體範例請參閱個別終端文檔。

- 在介面配置中新增了 `fileSearch`：
  - 啟用或停用聊天介面中的檔案搜尋按鈕。
  - 類似於 `webSearch` 選項，但控制的是檔案搜尋（透過工具使用 RAG API）。
  - 不會停用 [智慧體檔案搜尋能力](/docs/features/agents_zh_TW#檔案搜尋)。
  - 若未指定，預設值為 `true`。
  - 詳情請參閱 [介面物件結構 - fileSearch](/docs/configuration/librechat_yaml/object_structure/interface_zh_TW#filesearch)。

- 更新了 [模型規範文檔](/docs/configuration/librechat_yaml/object_structure/model_specs_zh_TW) 中的參數支援：
  - 增加了對 `disableStreaming`、`thinking`、`thinkingBudget`、`web_search` 以及其他參數的支援。
