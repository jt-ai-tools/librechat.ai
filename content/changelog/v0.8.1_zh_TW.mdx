## 🏞️ 亮點介紹

自 [v0.8.0](https://www.librechat.ai/changelog/v0.8.0_zh_TW) 以來的主要功能與改進：

### 🤖 智慧體轉接 (Agent Handoffs) (測試版)
現在可以配置智慧體在需要特定專業知識時，將對話控制權轉交給其他專門的智慧體。

### 📎 支援直接上傳至供應商
檔案現在可以直接上傳至支援的供應商並由其處理，不再需要中間的 OCR 或其他多模態處理步驟。

### 🖼️ 改進組件 (Artifact) 管理與分享
組件功能已獲得強化，包含版本控制能力、更好的 UI 互動與渲染支援，並且現在可以在共享的對話中檢視。

### 🤖 模型更新
- **Gemini 3** 支援與計費
- **GPT-5.1** 支援及其 codex 變體
- **Grok 4** 變體（fast, 4-1-fast, code-fast）及其內容費率
- **Claude Opus 4.5 與 Haiku 4.5** 及其代幣費率與窗口限制
- **智譜 AI GLM** 家族支援，具備完整的上下文窗口與計費
- 強化模型計費涵蓋範圍，包含 **DeepSeek R1/V3**、**通義千問 (Qwen) 家族**、**Amazon Nova Premier**、**Llama 3.3**、**gpt-5-pro** 等，並優化了模式匹配邏輯

### ⚙️ 其他顯著變動
- 支援 Langfuse 追蹤 (Tracing)
- 整合 Helicone AI 閘道器
- 支援 Bedrock 提示詞快取 (Prompt Caching)
- 在產生回應期間防止螢幕進入睡眠
- Anthropic 文件上傳現在可使用檔案名稱作為上下文
- 為叢集部署中的多伺服器實例協調實作了基於 Redis 的分佈式領導者選舉 (Leader Election)
- 現在可透過模型規範 (Model Specs) 配置是否在聊天輸入框渲染特定的工具標籤（檔案搜尋、MCP 伺服器等）
- 整個應用程式的無障礙性改進，提供更佳的鍵盤導覽、螢幕閱讀器支援並符合無障礙標準
- Helm Chart 增強：自定義部署標籤、註解、初始化容器、ConfigMaps 支援，以及額外的 RAG API 環境變數
- 重構 MCP 註冊系統，具備分佈式快取、主從協調、受超時保護的伺服器初始化以及按使用者的工具快取
