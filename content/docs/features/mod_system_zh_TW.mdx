## 自動審核系統 (選填)
自動審核系統使用評分機制來追蹤使用者的違規行為。當使用者執行如過度登入、註冊或傳送訊息等操作時，會累積違規分數。一旦達到設定的閾值，該使用者及其相關 IP 將被暫時封鎖。此系統透過監測並懲罰快速或可疑的活動來確保平台安全。

在生產環境中，您應該部署 Cloudflare 或其他 DDoS 防護方案來真正保護伺服器免受過量請求的攻擊，但這些變更在很大程度上能保護您免受單個或少數惡意行為者針對您的部署實例進行代理攻擊。

### 備註

- 使用快取 (Caching) 來進行基礎安全性與違規記錄（包含封鎖、並行訊息、超出速率限制等）。
    - 在不久的將來，我將為生產環境實例加入 **Redis** 支援，這可以輕鬆地注入當前的快取設定中。
- 超出任何速率限制器（登入/註冊/訊息）皆被視為違規，預設分數為 1 分。
- 非瀏覽器來源 (Non-browser origin) 也被視為違規。
- 每項違規的預設分數皆可配置。
- 啟用任何限制器及/或封鎖功能即會啟動快取/紀錄。
- 違規日誌可在數據資料夾中找到，該資料夾會在日誌紀錄開始時建立：`librechat/data`
  - **僅紀錄違規行為**
  - `violations.json` 追蹤每位使用者各項違規的總次數。
  - `logs.json` 記錄每位使用者的每項個別違規行為。
- 封鎖日誌儲存在 MongoDB 的 `logs` 集合中。它們是暫時性的，僅在封鎖期間內存在。
    - 如果您想手動移除封鎖，必須手動從資料庫中移除該紀錄並重啟伺服器。
    - 針對此項功能也計畫提供 **Redis** 支援。

### 速率限制器類型 (Rate Limiter Types)

#### 登入與註冊速率限制
透過限制在特定時間窗口內，單一 IP 位址可執行的登入嘗試或新帳號註冊次數，來防止暴力破解攻擊與垃圾郵件註冊。

#### 訊息速率限制
控制使用者傳送訊息的頻率，以防止垃圾訊息與濫用：
- **並行訊息限制 (Concurrent Message Limiting)**：限制使用者可以同時傳送多少則訊息（防止使用者開啟多個分頁來規避限制）。
- **訊息頻率限制 (Message Frequency Limiting)**：控制傳送訊息的間隔頻率，可針對 IP 位址或個別使用者進行配置。

#### 匯入對話速率限制
限制在特定時間窗口內可匯入的對話數量，防止對話匯入功能被濫用。這有助於防止：
- 大量數據匯入壓垮伺服器。
- 自動化腳本嘗試以匯入數據灌爆系統。
- 處理大量匯入導致的資源耗盡。

預設限制：
- 基於 IP：每分鐘 100 次匯入
- 基於使用者：每分鐘 50 次匯入（預設停用）

#### 對話分叉 (Forking) 速率限制
控制使用者建立現有對話副本（分叉）的頻率。這可以防止：
- 大量重複對話導致資料庫過度膨脹。
- 分叉操作導致的資源耗盡。
- 將分叉功能用於垃圾訊息或數據蒐集的濫用行為。

預設限制：
- 基於 IP：每分鐘 30 次分叉
- 基於使用者：每分鐘 7 次分叉（預設停用）

#### 檔案上傳速率限制
透過 `librechat.yaml` 檔案配置，控制使用者上傳檔案的頻率，以防止儲存空間濫用與頻寬耗盡。

#### 文字轉語音 (TTS) 速率限制
控制使用者請求文字轉語音轉換的頻率。這可以防止：
- 過高的 API 使用成本。
- 音訊產生導致的伺服器資源耗盡。
- 將 TTS 功能用於數據蒐集的濫用行為。

透過 `librechat.yaml` 檔案配置，支援針對每 IP 與每使用者自定義限制。

#### 語音轉文字 (STT) 速率限制
控制使用者提交音訊進行轉錄的頻率。這可以防止：
- 過高的 API 使用成本。
- 音訊處理導致的伺服器資源耗盡。
- 將 STT 功能用於未經授權轉錄服務的濫用行為。

透過 `librechat.yaml` 檔案配置，支援針對每 IP 與每使用者自定義限制。

#### 密碼重設速率限制
控制使用者請求密碼重設郵件的頻率。這可以防止：
- 郵件轟炸攻擊。
- 對密碼重設系統的濫用。
- 過度的電子郵件服務使用。

#### 郵件驗證速率限制
控制使用者請求郵件驗證訊息的頻率。這可以防止：
- 透過驗證系統進行垃圾郵件攻擊。
- 郵件服務濫用。
- 驗證請求導致的資源耗盡。

#### 工具呼叫速率限制
控制使用者進行工具/外掛程式呼叫的頻率。這可以防止：
- 整合工具產生的過高 API 使用量。
- 對外部服務整合的濫用。
- 工具處理導致的資源耗盡。

#### 對話存取速率限制
控制使用者存取或嘗試存取對話的頻率。這可以防止：
- 未經授權的存取嘗試。
- 數據抓取 (scraping) 攻擊。
- 過度的資料庫查詢。

### 速率限制器概覽

專案目前的速率限制器如下（預設值請參見下方的設定部分）：

- 登入與註冊速率限制
- `選填：` 並行訊息限制（每位使用者同一時間僅能傳送 X 則訊息）
- `選填：` 訊息頻率限制（使用者傳送訊息的間隔頻率，按 IP 與使用者配置）
- `選填：` 匯入對話限制（使用者匯入對話的頻率，按 IP 與使用者配置）
- `選填：` 對話分叉限制（使用者分叉對話的頻率，按 IP 與使用者配置）
- `選填：` 文字轉語音 (TTS) 限制（透過 [`librechat.yaml` 配置檔案](/docs/configuration/librechat_yaml/object_structure/config_zh_TW#ratelimits)配置）
- `選填：` 語音轉文字 (STT) 限制（透過 [`librechat.yaml` 配置檔案](/docs/configuration/librechat_yaml/object_structure/config_zh_TW#ratelimits)配置）
- `選填：` 檔案上傳限制（透過 [`librechat.yaml` 配置檔案](/docs/configuration/librechat_yaml/object_structure/config_zh_TW#ratelimits)配置）

**如需更多細節，請參閱此處提供的配置指南：[自動審核](/docs/configuration/mod_system_zh_TW)**
