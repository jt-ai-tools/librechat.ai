## 簡介

LibreChat 的代碼解釋器 (Code Interpreter) API 提供了一種安全且無煩惱的方式，透過簡單的 API 介面執行程式碼並管理檔案。無論您是透過 LibreChat 的智慧體使用，還是直接將其整合到您的應用程式中，此 API 都為多種程式語言提供了一個強大的沙盒執行環境。

## 快速入門

1. 造訪 [code.librechat.ai](https://code.librechat.ai/pricing) 獲取您的 API 金鑰。
2. 將 API 整合至您的應用程式或透過 LibreChat 使用。
3. 開始安全地執行程式碼並產生檔案。

## 核心特色

### 支援的語言

執行多種語言的程式碼：
- Python, Node.js (JS/TS), Go, C/C++, Java, PHP, Rust, Fortran, Rscript

### 無縫檔案處理

- 上傳檔案進行處理
- 下載產生的輸出結果
- 安全的檔案管理
- 基於工作階段 (session) 的檔案組織

### 安全與便利

- 安全的沙盒執行環境
- 無需本地安裝或配置
- 無需伺服器部署
- 無需管理配置

## 使用 API

### 在 LibreChat 中

此 API 在 LibreChat 中具備一等支援，主要透過兩種方式使用：

1. **[AI 智慧體 (AI Agents)](/docs/features/agents_zh_TW#代碼解釋器)**：在您的智慧體配置中啟用代碼解釋器，讓其能自動執行程式碼與處理檔案。

2. **手動執行**：點擊聊天介面中程式碼區塊旁的「執行程式碼」按鈕，如下圖所示：

![LibreChat 中的代碼解釋器](/images/agents/run_code.png)

### 設定 API 金鑰

- **按使用者設定**：在系統提示時（使用上述方法時）於 LibreChat 中輸入您的 API 金鑰。
- **全域設定**：在專案的 .env 檔案中使用 `LIBRECHAT_CODE_API_KEY` 環境變數（為所有使用者提供存取權限）。

### 直接 API 整合

代碼解釋器 API 可以透過簡單的 API 金鑰驗證整合至任何應用程式：

1. 從 [code.librechat.ai](https://code.librechat.ai/pricing) 獲取您的 API 金鑰。
2. 在您的請求標頭中包含 `x-api-key`。

### 企業版 (Enterprise)

企業版方案需要使用 `LIBRECHAT_CODE_BASEURL` 環境變數來對應自行代管的 API 實例，並搭配儀表板產生的 API 金鑰使用，用法與上述相同。

## 核心功能

### 程式碼執行

- 執行支援語言的程式碼片段
- 接收標準輸出 (stdout) 與標準錯誤 (stderr)
- 獲取執行統計數據（記憶體使用量、CPU 時間）
- 處理程式參數
- 存取執行狀態與結果

### 檔案操作

- 上傳輸入檔案
- 下載產生的輸出
- 列出可用檔案
- 刪除不必要的檔案
- 管理檔案工作階段

### 限制事項
- 程式碼無法存取網路。
- 每次執行僅能產生最多 10 個檔案。
- 執行限制視方案而異：
  - **Hobby (愛好者)**: 
    - 每次執行 256 MB RAM
    - 每個檔案上傳限制 25 MB
    - 每月 750 次請求
  - **Enthusiast (熱衷者)**: 
    - 每次執行 512 MB RAM
    - 每個檔案上傳限制 50 MB
    - 每月 3,000 次請求
  - **Pro (專業版)**: 
    - 每次執行 512 MB RAM
    - 每個檔案上傳限制 150 MB
    - 每月 7,000 次請求
- [企業版方案](https://code.librechat.ai/pricing)提供自定義的限制與功能。

## 使用情境

- **代碼測試**：測試多種語言的程式碼片段。
- **檔案處理**：透過程式化方式轉換與分析檔案。
- **AI 應用程式**：安全地執行 AI 產生的程式碼。
- **開發工具**：建構互動式的程式撰寫環境。
- **客觀邏輯**：驗證代碼邏輯與正確性，進而優化 AI 模型。

## 為什麼需要付費 API？

雖然 LibreChat 本身是遵循 MIT 授權條款的免費開源專案，但代碼解釋器 API 作為付費服務提供有幾個核心原因：

1. **專案永續性**：訂閱 API 方案能直接支持專案的開發，比 [GitHub Sponsors](https://github.com/sponsors/danny-avila) 更具實效。您的訂閱有助於確保 LibreChat 持續成長與改進。

2. **技術考量**：若在核心專案中內建程式碼執行能力，會大幅增加系統複雜度與硬體需求，並非所有使用者皆有此需求。API 服務消除了這些隱憂，同時維持核心應用程式的輕量化。

3. **代管服務的優勢**：
   - 無需複雜配置
   - 即刻可用
   - 定期更新與維護
   - 專業技術支援
   - 安全的沙盒環境

4. **智慧財產保護**：代碼解釋器的架構代表了安全、可擴展沙盒技術的重大創新。雖然存在類似解決方案，但往往缺乏讓此實作獨樹一格的全面安全措施與擴展特性。將此組件作為閉源 API 提供，有助於保護這些創新，並確保服務維持高標準的安全與效能。

即使您只是偶爾使用程式碼執行功能，訂閱也有助於支持 LibreChat 的開發，同時讓您享有專業級的功能體驗。這是一個雙贏的方案：讓核心專案保持免費，同時為有需要的用戶提供選用的進階能力。
