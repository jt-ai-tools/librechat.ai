感謝您有興趣協助翻譯 LibreChat！我們仰賴社群的貢獻，讓全球的使用者都能無障礙地使用此應用程式。所有的翻譯工作皆透過 [Locize](https://locize.com) 進行管理，這是一個功能強大且與我們專案無縫整合的翻譯管理系統。

## 翻譯運作機制

- **集中化管理：**
LibreChat 的所有翻譯字串都集中維護在 Locize 上。這種集中化管理確保了一致性，並簡化了整個應用程式的更新流程。

- **自動化更新：**
在 Locize 上所做的變更會自動同步至我們的專案。您可以透過我們儲存庫中的動態標籤來監控每種語言的翻譯進度。

- **社群驅動：**
我們歡迎各種語言的貢獻。您的協助將使 LibreChat 觸及更廣泛的受眾，並支援使用者以母語使用。

## 處理翻譯字串中的 `{{0}}` 與 `{{1}}`

有時翻譯字串需要包含動態內容。這些動態部分稱為 **插值 (interpolations)**，由雙大括號包圍的預留位置表示（例如：`{{0}}` 或 `{{1}}`）。翻譯此類字串時，務必在正確的位置保留這些預留位置。

以下提供兩個範例供您參考：

### 範例 1：單一插值

考慮翻譯鍵值 `com_assistants_completed_action`。原始英文文本為：

```text
Talked to {{0}}
```

對於中文翻譯，請確保預留位置完整且放置位置適當：

```text
與 {{0}} 進行了對話
```

下圖顯示了翻譯字串中單一插值的呈現方式：

![單一插值](https://github.com/user-attachments/assets/384ef7c1-9b02-490c-8ca4-b7f74943893f)

---

### 範例 2：多重插值

接著，看翻譯鍵值 `com_files_number_selected`，它包含兩個預留位置。英文版本為：

```text
{{0}} of {{1}} item(s) selected
```

在中文翻譯中，必須保留兩個預留位置並正確定位：

```text
已選取 {{0}} / {{1}} 個項目
```

下圖說明了翻譯字串中多重插值的出現方式：

![多重插值](https://github.com/user-attachments/assets/f3376487-e092-442a-b849-b6ab5d5b390d)

## 新增語言

如果您在我們的翻譯表中沒看到您的語言，您可以協助我們擴展語言支援：

1. **建立新 Issue：**
在我們的 GitHub 儲存庫開啟一個新 issue：[LibreChat Issues](https://github.com/danny-avila/LibreChat/issues)。

2. **選取新語言請求模板：**
使用 **New Language Request** 模板並提供：
 - 您語言的全名（例如：西班牙語、普通話）。
 - 您語言的 [ISO 639-1](https://www.w3schools.com/tags/ref_language_codes.asp) 代碼（例如：西班牙語為 `es`）。

3. **與維護者協作：**
我們的維護者將檢閱您的請求，並與您合作整合新語言。核准後，您的語言將出現在翻譯進度表中，您可以開始貢獻。
