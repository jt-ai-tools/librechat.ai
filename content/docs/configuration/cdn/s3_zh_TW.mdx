Amazon S3 是一種具備擴充性且安全的物件儲存服務，可作為 LibreChat 的檔案儲存後端。請遵循以下步驟來配置您的 S3 儲存桶。

## 1. 建立 AWS 帳號並配置 IAM 使用者（或使用 IRSA）

### 選項 A：使用具備明確憑證的 IAM 使用者

1. **登入 AWS：**
   - 開啟 [AWS 管理主控台 (AWS Management Console)](https://aws.amazon.com/console/) 並使用您的帳號登入。

2. **建立或使用現有的 IAM 使用者：**
   - 導覽至 **IAM (Identity and Access Management)** 區段。
   - 建立一個具備 **程式化存取 (Programmatic Access)** 權限的新 IAM 使用者，或選擇現有的使用者。
   - 附加適當的原則（例如：`AmazonS3FullAccess` 或具備限制 S3 權限的自定義原則）。
   - 建立使用者後，您將獲得 **AWS_ACCESS_KEY_ID** 與 **AWS_SECRET_ACCESS_KEY**。請務必妥善儲存。

### 選項 B：在 Kubernetes 中使用 IRSA (IAM Roles for Service Accounts)

如果您是在 Kubernetes（例如 EKS）上部署 LibreChat，可以使用 IRSA 將 AWS 權限指派給您的 Pod，而無需提供明確的憑證。使用 IRSA 的步驟如下：

1. **為您的 EKS 服務帳號建立信任原則 (Trust Policy)**（範例如下）：
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [
       {
         "Effect": "Allow",
         "Principal": {
           "Federated": "arn:aws:iam::{AWS_帳號}:oidc-provider/oidc.eks.us-east-1.amazonaws.com/id/{EKS_OIDC}"
         },
         "Action": "sts:AssumeRoleWithWebIdentity",
         "Condition": {
           "StringEquals": {
             "oidc.eks.us-east-1.amazonaws.com/id/{EKS_OIDC}:sub": "system:serviceaccount:librechat:librechat",
             "oidc.eks.us-east-1.amazonaws.com/id/{EKS_OIDC}:aud": "sts.amazonaws.com"
           }
         }
       }
     ]
   }
   ```
2. **建立一個授予必要 S3 權限的原則 (Policy)**（範例如下）：
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "VisualEditor0",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObjectAcl",
        "s3:GetObject",
        "s3:ListBucket",
        "s3:DeleteObject"
      ],
      "Resource": [
        "arn:aws:s3:::我的-範例-librechat-儲存桶/*",
        "arn:aws:s3:::我的-範例-librechat-儲存桶"
      ]
    }
  ]
}
   ```
3. **註解 (Annotate) 您的 Kubernetes ServiceAccount：**  
確保您的 LibreChat Pod 使用已針對 IRSA 進行註解的服務帳號。這樣一來，應用程式中的 AWS SDK（使用我們更新後的 S3 初始化代碼）將自動使用由 IRSA 提供的臨時憑證，而無需 AWS 憑證的環境變數。

## 2. 建立 S3 儲存桶 (Bucket)

1. **開啟 S3 主控台：**
- 前往 [Amazon S3 主控台](https://s3.console.aws.amazon.com/s3/)。

2. **建立新儲存桶：**
- 點擊 **"Create bucket"**（建立儲存桶）。
- **Bucket Name：** 輸入一個唯一的名稱（例如：`mylibrechatbucket`）。
- **Region：** 選擇距離您的使用者最近的 AWS 區域（例如：`us-east-1` 或 `eu-west-1`）。
- **配置選項：** 根據需求設定其他選項，然後點擊 **"Create bucket"**。

## 3. 更新您的環境變數

如果您 **不是** 使用 IRSA，請在專案根目錄的 `.env` 檔案中建立或更新下列配置：

```bash filename=".env"
AWS_ACCESS_KEY_ID=您的_access_key_id
AWS_SECRET_ACCESS_KEY=您的_secret_access_key
AWS_REGION=您選擇的區域
AWS_BUCKET_NAME=您的儲存桶名稱
AWS_ENDPOINT_URL=您的終端URL
# AWS_FORCE_PATH_STYLE=false
```

- **AWS_ACCESS_KEY_ID：** 您的 IAM 使用者存取金鑰。
- **AWS_SECRET_ACCESS_KEY：** 您的 IAM 使用者秘密金鑰。
- **AWS_REGION：** 您的 S3 儲存桶所在的 AWS 區域。
- **AWS_BUCKET_NAME：** 您建立的 S3 儲存桶名稱。
- **AWS_ENDPOINT_URL：** （選填）自定義 AWS 終端 URL。適用於相容 S3 的服務，如 MinIO、Cloudflare R2、Hetzner Object Storage 與 Backblaze B2。
- **AWS_FORCE_PATH_STYLE：** （選填）對於需要路徑樣式 URL (`endpoint/bucket/key`) 而非虛擬託管樣式 (`bucket.endpoint/key`) 的供應商，請設定為 `true`。Hetzner Object Storage、MinIO 以及其 SSL 憑證不支援儲存桶子網域的類似供應商需要此設定。AWS S3 或 Cloudflare R2 不需要。預設值：`false`。

如果您在 Kubernetes 上使用 **IRSA**，則 **不需要** 在環境中設定 `AWS_ACCESS_KEY_ID` 與 `AWS_SECRET_ACCESS_KEY`。AWS SDK 會自動透過指派給 Pod 的服務帳號獲取臨時憑證。請確保仍提供 `AWS_REGION` 與 `AWS_BUCKET_NAME`。

## 4. 配置 LibreChat 使用 Amazon S3

更新您的 LibreChat 配置檔案 (`librechat.yaml`)，指定應用程式應使用 Amazon S3 處理檔案：

```yaml filename="librechat.yaml"
version: 1.3.4
cache: true
fileStrategy: "s3"
```

<Callout type="warning" title="S3 預簽名 URL 會過期 —— 圖片與頭像將失效">
  S3 不透過 CDN 提供檔案。LibreChat 透過 **預簽名 URL (presigned URLs)** 存取 S3 檔案，這是一種具備可配置過期時間 (`S3_URL_EXPIRY_SECONDS`) 的臨時簽署權杖。AWS 將 IAM 使用者憑證的預簽名 URL 壽命限制為 7 天，而使用臨時憑證（STS/IAM 角色，如 IRSA）時僅為幾小時。一旦 URL 過期，其引用的圖片或頭像將在 UI 中顯示為損壞，直到重新整理頁面並產生新的 URL。

  LibreChat 中的重新整理邏輯套用並不一致 —— 例如，智慧體 (agent) 列表終端會返回儲存的（可能已過期的）URL，而單個智慧體終端則會重新整理它們。這會導致模型選擇器與聊天 UI 中出現明顯的頭像圖片損壞。詳情請參閱[相關討論](https://github.com/danny-avila/LibreChat/discussions/10280#discussioncomment-14803903)。

  **S3 非常適合文件儲存**（PDF、文字檔、代碼），這類場景適合使用短效的預簽名下載 URL。對於需要在 UI 中持久呈現的圖片與頭像，請使用 [Firebase](/docs/configuration/cdn/firebase_zh_TW) 或配置 `fileStrategies` 將該類型導向至具備 CDN 支援的策略：

  ```yaml filename="librechat.yaml"
  fileStrategies:
    avatar: "firebase"
    image: "firebase"
    document: "s3"
  ```

  CloudFront（Amazon 針對 S3 的 CDN，可完全消除預簽名 URL）目前正處於開發階段。
</Callout>

## 總結

1. **建立 AWS 帳號與 IAM 使用者（或配置 IRSA）：**
- 對於傳統部署，建立一個具備程式化存取權限的 IAM 使用者並獲取存取金鑰。
- 對於 Kubernetes 部署（例如在 EKS 上），設定 IRSA 以便您的 Pod 自動獲取臨時憑證。

2. **建立 S3 儲存桶：**
- 使用 Amazon S3 主控台建立儲存桶，選擇唯一的名稱與區域。

3. **更新環境變數：**
- 非 IRSA：在 `.env` 檔案中設定 `AWS_ACCESS_KEY_ID`、`AWS_SECRET_ACCESS_KEY`、`AWS_REGION` 與 `AWS_BUCKET_NAME`。
- IRSA：僅設定 `AWS_REGION` 與 `AWS_BUCKET_NAME`；確保 Pod 的服務帳號已正確註解。

4. **配置 LibreChat：**
- 在 `librechat.yaml` 配置檔案中將 `fileStrategy` 設定為 `"s3"`。

完成這些步驟後，您的 LibreChat 應用程式將使用 Amazon S3 來處理檔案的上傳、下載與刪除。此外，透過 IRSA 支援，您的應用程式可以在 Kubernetes 上安全執行，而無需嵌入長期的 AWS 憑證。

<Callout type="info" title="備註">
  請務必確保您的 AWS 憑證安全。切勿將其提交至公開儲存庫。請根據需要調整 IAM 原則，遵循最小權限原則。
</Callout>
