Firebase Storage 與 Firebase Hosting 的全球 CDN 整合，允許您透過分佈在世界各地的邊緣節點來提供儲存在 Firebase Storage 中的檔案。這使其成為 LibreChat 中唯一具備真實 CDN 支援的檔案儲存選項。

## 設定 Firebase 的步驟

1. 開啟 [Firebase 網站](https://firebase.google.com/)。
2. 點擊 「Get started」 (立即開始)。
3. 使用您的 Google 帳號登入。

- 將所有的 `firebaseConfig` 變數填入 `.env` 檔案中：

```bash

FIREBASE_AUTH_DOMAIN=auth_domain #來自 authDomain
FIREBASE_PROJECT_ID=project_id #來自 projectId
FIREBASE_STORAGE_BUCKET=storage_bucket #來自 storageBucket
FIREBASE_MESSAGING_SENDER_ID=messaging_sender_id #來自 messagingSenderId

```

    ```bash

    service firebase.storage {
      match /b/{bucket}/o {
        match /images/{userId}/{fileName} {
          allow read, write: if true;
        }
      }
    }
    ```

### 在 `librechat.yaml` 中配置 `fileStrategy`

最後，要讓應用程式啟用 Firebase，您必須在 `librechat.yaml` 配置檔案中設定下列內容：

```yaml
  version: 1.3.4
  cache: true
  fileStrategy: "firebase"
```

如需更多關於 `librechat.yaml` 配置檔案的資訊，請參閱此處的指南：[自定義終端與配置](/docs/configuration/librechat_yaml/index_zh_TW)。

<Callout type="warning" title="使用 Firebase 時將對話匯出為 PNG">

### Firebase Storage 設定 CORS 的逐步指南

---

#### **第一步：建立 CORS 配置檔案**

- 開啟您偏好的文字編輯器。
- 建立一個新檔案並命名為 `cors.json`。
- 新增下列配置以允許來自 「https://ai.example.com」 的存取：

```json
[
  {
    "origin": ["https://ai.example.com"],
    "method": ["GET", "POST", "DELETE", "PUT"],
    "maxAgeSeconds": 3600
  }
]
```
- 儲存檔案。

---

#### **第二步：套用 CORS 配置**

- 開啟您的終端機或命令提示字元。
- 導覽至儲存 `cors.json` 檔案的目錄。
- 執行下列命令，將 `<您的儲存空間儲存桶名稱>` 替換為您的 Firebase Storage 儲存桶名稱：

```shell
gsutil cors set cors.json gs://<您的儲存空間儲存桶名稱>
```

---

#### **第三步：驗證 CORS 設定**

- 若要確認 CORS 設定是否已正確套用，您可以使用下列命令獲取當前的 CORS 配置：

```shell
gsutil cors get gs://<您的儲存空間儲存桶名稱>
```

- 輸出結果應反映您在 `cors.json` 檔案中指定的設定。

---

#### **第四步：測試配置**

- 嘗試從允許的來源（「https://ai.example.com」）將對話匯出為 PNG。
- 如果一切設定正確，您應該不會遇到任何 CORS 問題。

---

> **備註：** 請務必確保您僅針對受信任的來源套用 CORS 設定，以維護應用程式的安全性。請根據您的具體需求調整允許的方法與標頭。

---

大功告成！您已成功為您的 Firebase Storage 儲存桶配置了 CORS，以允許來自特定來源的請求。請記住，套用配置時請將 `<您的儲存空間儲存桶名稱>` 替換為實際的名稱，並將 `https://ai.example.com` 替換為您自己的網域。

</Callout>
