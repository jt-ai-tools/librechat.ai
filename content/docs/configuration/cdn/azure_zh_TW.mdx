## 生產環境設定

Azure Blob Storage 為 LibreChat 中的檔案提供具備擴充性且安全的物件儲存服務。請遵循以下步驟來配置您的 Azure Blob Storage。

## 1. 建立 Azure 儲存體帳戶

1. **登入 Azure：**
   - 開啟 [Azure 入口網站 (Azure Portal)](https://portal.azure.com/) 並使用您的 Microsoft 帳號登入。

2. **建立儲存體帳戶：**
   - 點擊 **"Create a resource"** (建立資源) 並搜尋 **"Storage account"** (儲存體帳戶)。
   - 點擊 **"Create"** 並填寫必要的詳細資訊：
     - **Subscription & Resource Group (訂閱與資源群組)：** 選擇您的訂閱，並選擇現有的資源群組或建立一個新群組。
     - **Storage Account Name (儲存體帳戶名稱)：** 輸入一個唯一的名稱（例如：`mylibrechatstorage`）。
     - **Region (區域)：** 選擇最接近您的使用者的區域。
     - **Performance & Redundancy (效能與備援)：** 選擇最符合您需求的效能層級與備援等級。
   - 點擊 **"Review + Create"**，接著點擊 **"Create"**。等待部署完成。

## 2. 設定身份驗證

您可以選擇兩種方式來與您的 Azure 儲存體帳戶進行身份驗證：

### 選項 A：使用連接字串 (Connection String)

1. **導覽至存取金鑰：**
   - 在您新建立的儲存體帳戶中，前往側邊欄的 **"Access keys"** (存取金鑰)。

2. **複製連接字串：**
   - 複製其中一個提供的連接字串。此字串包含連接到您的 Blob 儲存體帳戶所需的憑證。

### 選項 B：使用受控識別 (Managed Identity)

如果您的 LibreChat 應用程式執行於支援受控識別的 Azure 服務（如 Azure VM、App Service 或 AKS），您可以使用受控識別來代替連接字串。

1. **指派受控識別：**
   - 確保您的 Azure 資源（VM、App Service 或 AKS）已啟用系統指派或使用者指派的受控識別。

2. **授與儲存權限：**
   - 在您的儲存體帳戶中，將 **Storage Blob Data Contributor** (儲存體 Blob 資料參與者)（或類似範圍的角色）指派給您的受控識別。這允許您的應用程式在不使用連接字串的情況下存取 Blob 儲存體。

## 3. 更新您的環境變數

在專案根目錄的 `.env` 檔案中建立或更新下列配置：

```bash filename=".env"
# 選項 A：使用連接字串
AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;AccountName=您的帳戶名稱;AccountKey=您的帳戶金鑰;EndpointSuffix=core.windows.net

# 選項 B：使用受控識別（若使用受控識別，請勿設定連接字串）
AZURE_STORAGE_ACCOUNT_NAME=您的帳戶名稱

AZURE_STORAGE_PUBLIC_ACCESS=false
AZURE_CONTAINER_NAME=files
```

- **AZURE_STORAGE_CONNECTION_STRING：** 如果您使用選項 A，請設定此項。
- **AZURE_STORAGE_ACCOUNT_NAME：** 如果您使用選項 B（受控識別），請設定此項。請勿同時設定兩者。
- **AZURE_STORAGE_PUBLIC_ACCESS：** 如果您不希望 Blob 預設為公開存取，請設定為 `false`。如果您需要公開存取（例如公開可見的圖片），請設定為 `true`。
- **AZURE_CONTAINER_NAME：** 這是您的應用程式將使用的容器名稱（例如：`files`）。如果該容器不存在，應用程式將自動建立它。

## 4. 配置 LibreChat 使用 Azure Blob Storage

更新您的 LibreChat 配置檔案 (`librechat.yaml`)，指定應用程式應使用 Azure Blob Storage 處理檔案：

```yaml filename="librechat.yaml"
version: 1.3.4
cache: true
fileStrategy: "azure_blob"
```

<Callout type="warning" title="Azure Blob Storage 是物件儲存，而非 CDN">
  Azure Blob Storage 直接從來源端存取並提供檔案 —— 它不是 CDN。為了獲得最佳效能與全球傳遞，圖片與頭像最好透過 CDN 提供。目前，[Firebase](/docs/configuration/cdn/firebase_zh_TW) 是唯一具備 CDN 支援的儲存選項。

  您可以使用 `fileStrategies` 將頭像與圖片導向至 Firebase，同時將文件保留在 Azure Blob Storage 中：

  ```yaml filename="librechat.yaml"
  fileStrategies:
    avatar: "firebase"
    image: "firebase"
    document: "azure_blob"
  ```
</Callout>

## 總結

1. **建立儲存體帳戶：**  
登入 Azure 入口網站，建立儲存體帳戶，並等待部署完成。

2. **設定身份驗證：**
- **選項 A：** 從儲存體帳戶的 **"Access keys"** 中獲取連接字串。
- **選項 B：** 在您的 Azure 資源上啟用受控識別，並授與適當的儲存權限。

3. **更新環境變數：**  
在您的 `.env` 檔案中，設定下列其中一項：
- `AZURE_STORAGE_CONNECTION_STRING`（用於選項 A），或
- `AZURE_STORAGE_ACCOUNT_NAME`（用於選項 B），以及：
- `AZURE_STORAGE_PUBLIC_ACCESS` 和
- `AZURE_CONTAINER_NAME`。

4. **配置 LibreChat：**  
在 `librechat.yaml` 配置檔案中將 `fileStrategy` 設定為 `"azure_blob"`。

完成這些步驟後，您的 LibreChat 應用程式將自動建立容器（如果尚不存在），並使用 Azure Blob Storage 管理檔案的上傳、下載與刪除。受控識別透過消除對長期憑證的需求，提供了一個安全的替代方案。

## 使用 Azurite 進行本地開發

對於本地開發與測試，您可以使用 [Azurite](https://github.com/Azure/Azurite)，這是一個 Azure 儲存體模擬器，提供了一個本地環境來測試您的 Azure Blob Storage 整合，而無需實際的 Azure 帳號。

### 1. 設定 Azurite

您可以透過多種方式執行 Azurite：

#### 選項 A：使用 VS Code 擴充功能（推薦開發使用）

1. 在 VS Code 中安裝 [Azurite 擴充功能](https://marketplace.visualstudio.com/items?itemName=Azurite.azurite)。
2. 開啟指令面板 (Ctrl+Shift+P 或 Cmd+Shift+P)。
3. 搜尋並選擇 「Azurite: Start」。

這將以預設設定在背景啟動 Azurite。

#### 選項 B：使用 Docker

```bash
docker run -p 10000:10000 -p 10001:10001 -p 10002:10002 mcr.microsoft.com/azure-storage/azurite
```

#### 選項 C：使用 npm

```bash
npm install -g azurite
azurite --silent --location /path/to/azurite/workspace --debug /path/to/debug/log
```

### 2. 為本地開發配置環境變數

將下列環境變數新增至您的 `.env` 檔案中：

```bash filename=".env"
# 本地開發用的 Azurite 連接字串
AZURE_STORAGE_CONNECTION_STRING="DefaultEndpointsProtocol=http;AccountName=devstoreaccount1;AccountKey=Eby8vdM02xNOcqFlqUwJPLlmEtlCDXJ1OUzFT50uSRZ6IFsuFq2UVErCz4I6tq/K1SZFPTOtr/KBHBeksoGMGw==;BlobEndpoint=http://127.0.0.1:10000/devstoreaccount1;"
AZURE_STORAGE_PUBLIC_ACCESS=true
AZURE_CONTAINER_NAME=files
```

備註：
- `AccountKey` 的值是 Azurite 使用的預設開發金鑰。
- 連接使用 `http` 協定而非本地開發用的 `https`。
- `BlobEndpoint` 指向在 10000 連接埠上運行的本地 Azurite 實例。

### 3. 驗證連線

要驗證您的應用程式是否能連接到本地 Azurite 實例：

1. 啟動您的 LibreChat 應用程式。
2. 嘗試透過介面上傳檔案。
3. 檢查 Azurite 日誌以確認連線與操作。

如果您使用的是 VS Code 擴充功能，可以透過從下拉選單選擇 「Azurite Blob」 在輸出 (Output) 面板中查看 Azurite 日誌。

<Callout type="info" title="備註">
  預設的 Azurite 帳戶金鑰是固定值，僅供開發用途。切勿在生產環境中使用此金鑰。請務必確保您的連接字串安全，且切勿將其提交至公開儲存庫。
</Callout>
