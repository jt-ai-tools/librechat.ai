歡迎閱讀透過 `.env` 檔案配置應用程式環境的全面指南。本文件是您瞭解與自定義環境變數的一站式資源，這些變數將決定應用程式在不同情境下的行為。

雖然預設設定為標準的 `docker` 安裝提供了堅實的基礎，但深入研讀本指南將揭示 LibreChat 的全部潛力。本指南賦予您根據精確需求客製化 LibreChat 的能力。瞭解如何調整語言模型的可用性、整合社群登入、管理自動審核系統等等。這一切都是為了讓您能夠微調 LibreChat，以獲得最佳的使用者體驗。

> **提醒：變更配置後，請重啟 LibreChat 以使設定生效**

或者，您可以在與 LibreChat 主 `docker-compose.yml` 檔案相同的目錄中建立一個名為 `docker-compose.override.yml` 的新檔案。在該檔案的 `environment` 欄位下，您可以根據需要設定 .env 變數，或修改主 `docker-compose.yml` 提供的預設配置，而無需直接編輯或複製整個原始檔案。

如需更多資訊，請參閱：

- 我們的快速指南：
    - **[Docker 覆寫 (Docker Override)](/docs/configuration/docker_override_zh_TW)**

- Docker 官方文件：
    - **[docker docs -瞭解多個 compose 檔案](https://docs.docker.com/compose/multiple-compose-files/extends/#understanding-multiple-compose-files)**
    - **[docker docs - 合併 compose 檔案](https://docs.docker.com/compose/multiple-compose-files/merge/#merge-compose-files)**
    - **[docker docs - 指定多個 compose 檔案](https://docs.docker.com/compose/reference/#specifying-multiple-compose-files)**

- 您也可以在 LibreChat 資料夾或 GitHub 上查看 LibreChat 的覆寫檔案範例：
    - **[docker-compose.override.example](https://github.com/danny-avila/LibreChat/blob/main/docker-compose.override.yml.example)**

### 連接埠 (Port)

- 伺服器在特定連接埠上監聽。
- `PORT` 環境變數設定伺服器監聽的連接埠。預設值為 `3080`。

### 信任代理 (Trust proxy)
使用距離 Express 應用程式最多 n 個躍點（hops）的位址。
req.socket.remoteAddress 是第一個躍點，其餘躍點則從 X-Forwarded-For 標頭中由右至左搜尋。
數值為 0 代表第一個不被信任的位址將是 req.socket.remoteAddress，即沒有反向代理。
`TRUST_PROXY` 環境變數的預設值為 `1`。

如需更多資訊，請參閱 [Express.js - trust proxy](https://expressjs.com/en/guide/behind-proxies.html)。

### 靜態檔案處理 (Static File Handling)

<OptionTable
  options={[
    ['STATIC_CACHE_MAX_AGE', 'string', 'Cache-Control max-age（秒）','STATIC_CACHE_MAX_AGE=172800'],
    ['STATIC_CACHE_S_MAX_AGE', 'string', '共享快取（CDN 與代理）的 Cache-Control s-maxage（秒）','STATIC_CACHE_S_MAX_AGE="86400"'],
    ['DISABLE_COMPRESSION', 'boolean', '停用靜態檔案的壓縮。','DISABLE_COMPRESSION=false'],
    ['ENABLE_IMAGE_OUTPUT_GZIP_SCAN', 'boolean', '如果同一資料夾中存在上傳圖片的 gzipped 版本，則啟用對其的提供。','ENABLE_IMAGE_OUTPUT_GZIP_SCAN=true'],
  ]}
/>

**行為說明：**

設定靜態檔案的 [Cache-Control](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Cache-Control) 標頭。這些配置僅在 `NODE_ENV` 設定為 `production` 時觸發。

* 取消註釋 `STATIC_CACHE_MAX_AGE` 可更改靜態檔案的本地 `max-age`。預設為 2 天（172800 秒）。
* 取消註釋 `STATIC_CACHE_S_MAX_AGE` 可設定共享快取（CDN 與代理）的 `s-maxage`。預設為 1 天（86400 秒）。
* 取消註釋 `DISABLE_COMPRESSION` 可停用靜態檔案壓縮。預設為啟用壓縮。
* 取消註釋 `ENABLE_IMAGE_OUTPUT_GZIP_SCAN` 可啟用掃描並提供圖片的 gzipped 版本（前提是它們已預先壓縮在同一資料夾中，具有相同名稱且帶有 .gz 副檔名）。預設情況下，停用上傳圖片的 gzip 掃描。

<Callout type="warning" title="警告">
- 這僅影響由 API 伺服器提供的靜態檔案，不適用於 _Firebase_、_NGINX_ 或任何其他配置。
</Callout>

### MongoDB 資料庫

<OptionTable
  options={[
    ['MONGO_URI', 'string', '指定 MongoDB URI。','MONGO_URI=mongodb://127.0.0.1:27017/LibreChat'],

如果不同，請將此更改為您的 MongoDB URI。您應該在 URI 中加入 `LibreChat` 或您自己的 `APP_TITLE` 作為資料庫名稱。

如果您使用線上資料庫，URI 格式為 `mongodb+srv://<使用者名稱>:<密碼>@<主機>/<資料庫>?<選項>`。您的 `MONGO_URI` 應該看起來像這樣：
* `mongodb+srv://username:password@host.mongodb.net/LibreChat?retryWrites=true` （在使用線上資料庫時，`retryWrites` 是您唯一需要的選項。）

### 防止公開搜尋引擎索引 (Prevent Public Search Engines Indexing)

預設情況下，您的網站不會被公開搜尋引擎（如 Google、Bing 等）索引。這意味著人們無法透過這些搜尋引擎找到您的網站。如果您希望提高網站的能見度與可搜尋性，可以將以下設定更改為 `false`。

<OptionTable
  options={[
    ['NO_INDEX', 'boolean', '防止公開搜尋引擎索引您的網站。', 'NO_INDEX=true'],
  ]}
/>

❗**注意：** 此方法並不保證對所有搜尋引擎都有效，某些搜尋引擎可能仍會為了快取或存檔等其他目的索引您的網站或網頁。因此，您不應僅依賴此方法來保護網站或網頁上的敏感或機密資訊。

### 配置驗證 (Configuration Validation)

預設情況下，如果 `librechat.yaml` 配置檔案包含驗證錯誤，LibreChat 將以錯誤（結束代碼 1）結束執行。這種「快速失敗」(fail-fast) 行為有助於在部署階段儘早發現配置問題，並防止以非預期的預設設定執行。

<OptionTable
  options={[
    ['CONFIG_BYPASS_VALIDATION', 'boolean', '設定為 `true` 時，即使 `librechat.yaml` 有驗證錯誤，伺服器仍會紀錄警告並繼續以預設配置啟動。這保留了舊版行為。', '# CONFIG_BYPASS_VALIDATION=true'],
  ]}
/>

<Callout type="warning" title="警告">
不建議在生產環境中使用 `CONFIG_BYPASS_VALIDATION=true`。這僅用於在偵錯配置問題時作為臨時解決方案。請務必修正配置檔案中的驗證錯誤。
</Callout>

<Callout type="warning" title="警告">
