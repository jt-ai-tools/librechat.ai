`webSearch` 配置讓您可以自定義 LibreChat 內的網頁搜尋功能，包括搜尋供應商、內容爬蟲 (scrapers) 與結果重新排序器 (rerankers)。

## 概覽

網頁搜尋功能由三個主要組件組成：

1. **搜尋供應商 (Search Providers)**：執行初始網頁搜尋的服務。
2. **爬蟲 (Scrapers)**：從網頁中提取內容的服務。
3. **重新排序器 (Rerankers)**：對搜尋結果進行重新排序以提高相關性的服務。

## 範例

```yaml filename="webSearch"
webSearch:
  # 搜尋供應商配置
  serperApiKey: "${SERPER_API_KEY}"
  searxngInstanceUrl: "${SEARXNG_INSTANCE_URL}"
  searxngApiKey: "${SEARXNG_API_KEY}"
  searchProvider: "serper" # 選項："serper"、"searxng"

  # 爬蟲配置
  firecrawlApiKey: "${FIRECRAWL_API_KEY}"
  firecrawlApiUrl: "${FIRECRAWL_API_URL}"
  firecrawlVersion: "${FIRECRAWL_VERSION}"
  scraperProvider: "firecrawl" # 選項："firecrawl"、"serper"

  # 重新排序器配置
  jinaApiKey: "${JINA_API_KEY}"
  jinaApiUrl: "${JINA_API_URL}"
  cohereApiKey: "${COHERE_API_KEY}"
  rerankerType: "jina" # 選項："jina"、"cohere"

  # 一般設定
  scraperTimeout: 7500 # 爬蟲請求的超時時間（毫秒，預設：7500）
  safeSearch: 1 # 選項：0 (關閉), 1 (中等 - 預設), 2 (嚴格)
```

**範例：**
```yaml filename="webSearch"
webSearch:
  firecrawlApiKey: "${FIRECRAWL_API_KEY}"
  firecrawlOptions:
    formats: ["markdown", "rawHtml"]
    includeTags: ["main", "article", ".content"]
    excludeTags: ["nav", "footer", ".ads"]
    waitFor: 2000
    timeout: 10000
    mobile: false
    blockAds: true
    onlyMainContent: true
    location:
      country: "US"
      languages: ["en"]
```

**備註：** 如需 Firecrawl 爬蟲選項與預設值的詳細資訊，請參閱 [Firecrawl API 文檔](https://docs.firecrawl.dev/api-reference/endpoint/scrape)。

## 備註

- API 金鑰可透過兩種方式配置：
  1. 設定 YAML 配置中指定的環境變數。
  2. 若未設定環境變數，系統會提示使用者透過 UI 提供 API 金鑰。
- 配置支援為每個組件（供應商、爬蟲、重新排序器）設定多個服務。
- 若未指定特定的服務類型，系統將嘗試該類別下所有可用的服務。
- 安全搜尋提供三個等級的內容過濾：關閉 (0)、中等 (1) 與嚴格 (2)。
- **切勿**將實際的 API 金鑰放入 YAML 配置中 —— 僅能使用環境變數名稱。

## 設定 SearXNG

SearXNG 是一款專注於隱私、可自行代管的元搜尋引擎。
更多資訊請參閱 [SearXNG 官方文檔](https://docs.searxng.org/)。

以下是為 LibreChat 設定自有的 SearXNG 實例的步驟：

### 使用 Docker Desktop

1. **搜尋官方 SearXNG 映像**
   - 開啟 Docker Desktop
   - 在 **Images** 標籤中搜尋 `searxng/searxng`
   - 點擊官方映像上的 **Run** 以自動下載並執行該映像的容器

2. **執行容器**
   - 下載完成後，在出現的面板中展開 **Optional Settings** 下拉選單
   - 設定您想要的配置細節（連接埠號、容器名稱等）
   - 點擊 **Run** 啟動容器

3. **為 LibreChat 配置 SearXNG**
   - 導覽至 Docker Desktop 中的 `Files` 標籤
   - 前往 `/etc/searxng/settings.yaml`
   - 開啟檔案編輯器
   - 導覽至 `formats` 區段
   - 加入 `json` 作為可接受的格式，以便 LibreChat 能與您的實例通訊
   - 儲存檔案

4. **重啟容器**
   - 重啟容器以使變更生效

您可以在 LibreChat 的 UI 內或透過 `librechat.yaml` 配置 SearXNG。

#### UI 配置

1. **開啟聊天輸入列中的工具下拉選單**
![工具配置按鈕](/images/web-search/tools_highlight.png)

2. **點擊「網頁搜尋 (Web Search)」旁邊的齒輪圖標**
![工具配置區段](/images/web-search/tools_config_highlight.png)

3. **從搜尋供應商下拉選單中選擇 SearXNG**
![SearXNG 下拉選單選取](/images/web-search/searxng_dropdown_highlight.png)

4. **輸入您的配置細節（例如實例 URL、爬蟲類型等）並點擊儲存**
![儲存網頁搜尋配置](/images/web-search/web_search_save.png)

5. **點擊工具下拉選單中的「網頁搜尋」選項**
![聊天介面中的網頁搜尋標籤](/images/web-search/websearch_highlight.png)
