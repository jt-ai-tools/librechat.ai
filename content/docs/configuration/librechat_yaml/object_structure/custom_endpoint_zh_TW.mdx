`custom` 陣列中的每個終端都應具備下列結構：

## 範例 

```yaml filename="終端物件結構"
endpoints:
    custom:
        # 使用 Mistral AI API 的範例
    - name: "Mistral"
        apiKey: "${YOUR_ENV_VAR_KEY}"
        baseURL: "https://api.mistral.ai/v1"
        models: 
        default: ["mistral-tiny", "mistral-small", "mistral-medium", "mistral-large-latest"]
        titleConvo: true
        titleModel: "mistral-tiny" 
        modelDisplayLabel: "Mistral"
        # addParams:
        #   safe_prompt: true # Mistral 特有的訊息審核值
        # 注意：對於 Mistral，必須刪除下列參數，否則會遇到 422 錯誤：
        dropParams: ["stop", "user", "frequency_penalty", "presence_penalty"]
```

或
```yaml filename="endpoints / custom / baseURL"

```
或
```yaml filename="endpoints / custom / baseURL"

```

**備註：**

* 如果您設定的 `baseURL` 是完整的補全 (completions) 終端位址，可以將 [directEndpoint](#directendpoint) 欄位設定為 `true` 以直接使用。
  - 這是必要的，因為應用程式預設會在 `baseURL` 後方附加 「/chat/completions」 或 「/completion」。

## titleMethod (標題產生方法)

**Key：**
<OptionTable
  options={[
    ['titleMethod', '字串', '控制產生對話標題所使用的方法。', '有效值："completion" (預設), "structured", "functions" ("structured" 的舊版別名)'],
  ]}
/>

**預設值：** `"completion"`

**可用方法：**
- **`"completion"`** - 使用標準補全 API，不包含工具/函式。與大多數 LLM 相容。
- **`"structured"`** - 使用結構化輸出產生標題。需要供應商/模型支援。
- **`"functions"`** - "structured" 的舊版別名。功能完全相同。

**範例：**
```yaml filename="endpoints / custom / titleMethod"
titleMethod: "completion"
```

## titlePrompt (標題提示詞)

**Key：**
<OptionTable
  options={[
    ['titlePrompt', '字串', '標題產生的自定義提示詞。必須包含 {convo} 佔位符。', '允許完整控制標題產生的方式。'],
  ]}
/>

**預設值：** 
```
分析此對話並提供：
1. 偵測到的對話語言
2. 使用偵測到的語言提供一個精簡標題（5 個字以內，不含標點符號或引號）

{convo}
```

**備註：**
- 必須始終包含 `{convo}` 佔位符。
- `{convo}` 佔位符將被替換為格式化後的對話內容。

**範例：**
```yaml filename="endpoints / custom / titlePrompt"
titlePrompt: "請為下列對話建立一個簡短、具描述性的標題：

{convo}

標題："
```

## titleEndpoint (標題產生終端)

**Key：**
<OptionTable
  options={[
    ['titleEndpoint', '字串', '指定用於標題產生的替代終端。', '允許使用不同的模型/終端來產生標題。'],
  ]}
/>

**預設值：** 使用當前的自定義終端

**接受的值：**
- `openAI`
- `azureOpenAI`
- `google`
- `anthropic`
- `bedrock`
- 其他自定義終端名稱

**範例：**
```yaml filename="endpoints / custom / titleEndpoint"
# 使用不同的自定義終端產生標題
endpoints:
  custom:
    - name: "我的聊天終端"
      apiKey: "${CHAT_API_KEY}"
      baseURL: "https://api.example.com/v1/chat"
      models:
        default: ["gpt-4"]
      titleEndpoint: "我的標題終端"
      
    - name: "我的標題終端"
      apiKey: "${TITLE_API_KEY}"
      baseURL: "https://api.example.com/v1/title"
      models:
        default: ["gpt-3.5-turbo"]
```

## headers (標頭)

**Key：**
<OptionTable
  options={[
    ['headers', '物件/字典', '為請求新增額外標頭。支援使用 `{{LIBRECHAT_USER_*}}` 進行動態使用者欄位替換、使用 `{{LIBRECHAT_BODY_*}}` 進行請求主體替換，以及使用 `${ENV_VAR}` 引用環境變數。', '`headers` 物件指定請求的自定義標頭。適用於身份驗證與設定內容類型。'],
  ]}
/>

**範例：**
```yaml filename="endpoints / custom / headers"
headers:
  x-api-key: "${ENVIRONMENT_VARIABLE}"
  Content-Type: "application/json"
  X-User-ID: "{{LIBRECHAT_USER_ID}}"
  X-User-Email: "{{LIBRECHAT_USER_EMAIL}}"
```
**備註：** 支援動態環境變數值，格式為：`"${VARIABLE_NAME}"`。


**可用的使用者欄位預留位置：**

| 預留位置 | 使用者欄位 | 類型 | 說明 |
|------------|------------|------|-------------|
| `{{LIBRECHAT_USER_ID}}` | `id` | 字串 | 使用者的唯一識別碼 |
| `{{LIBRECHAT_USER_NAME}}` | `name` | 字串 | 使用者的顯示名稱 |
| `{{LIBRECHAT_USER_USERNAME}}` | `username` | 字串 | 使用者的使用者名稱 |
| `{{LIBRECHAT_USER_EMAIL}}` | `email` | 字串 | 使用者的電子郵件地址 |
| `{{LIBRECHAT_USER_PROVIDER}}` | `provider` | 字串 | 身份驗證供應商 (例如: "email", "google", "github") |
| `{{LIBRECHAT_USER_ROLE}}` | `role` | 字串 | 使用者的角色 (例如: "user", "admin") |
| `{{LIBRECHAT_USER_GOOGLEID}}` | `googleId` | 字串 | Google 帳號 ID |
| `{{LIBRECHAT_USER_FACEBOOKID}}` | `facebookId` | 字串 | Facebook 帳號 ID |
| `{{LIBRECHAT_USER_OPENIDID}}` | `openidId` | 字串 | OpenID 帳號 ID |
| `{{LIBRECHAT_USER_SAMLID}}` | `samlId` | 字串 | SAML 帳號 ID |
| `{{LIBRECHAT_USER_LDAPID}}` | `ldapId` | 字串 | LDAP 帳號 ID |
| `{{LIBRECHAT_USER_GITHUBID}}` | `githubId` | 字串 | GitHub 帳號 ID |
| `{{LIBRECHAT_USER_DISCORDID}}` | `discordId` | 字串 | Discord 帳號 ID |
| `{{LIBRECHAT_USER_APPLEID}}` | `appleId` | 字串 | Apple 帳號 ID |
| `{{LIBRECHAT_USER_EMAILVERIFIED}}` | `emailVerified` | 布林值 → 字串 | 電子郵件驗證狀態 ("true" 或 "false") |
| `{{LIBRECHAT_USER_TWOFACTORENABLED}}` | `twoFactorEnabled` | 布林值 → 字串 | 兩步驟驗證狀態 ("true" 或 "false") |
| `{{LIBRECHAT_USER_TERMSACCEPTED}}` | `termsAccepted` | 布林值 → 字串 | 服務條款接受狀態 ("true" 或 "false") |

**可用的請求主體預留位置：**

| 預留位置 | 主體欄位 | 類型 | 說明 |
|------------|------------|------|-------------|
| `{{LIBRECHAT_BODY_CONVERSATIONID}}` | `conversationId` | 字串 | 當前對話識別碼 |
| `{{LIBRECHAT_BODY_PARENTMESSAGEID}}` | `parentMessageId` | 字串 | 父訊息識別碼 |
| `{{LIBRECHAT_BODY_MESSAGEID}}` | `messageId` | 字串 | 當前訊息識別碼 |

**使用請求主體預留位置的範例：**

```yaml filename="包含主體預留位置的 endpoints / custom / headers"
headers:

  X-Parent-Message-ID: "{{LIBRECHAT_BODY_PARENTMESSAGEID}}"
  X-Message-ID: "{{LIBRECHAT_BODY_MESSAGEID}}"
```
