## 概覽

`memory` 物件讓您可以為應用程式配置對話記憶與個人化功能。此配置控制了系統如何記憶與個人化對話，包括代幣 (token) 限制、訊息上下文窗口以及基於智慧體的記憶處理。

## 範例

```yaml filename="memory"
memory:
  disabled: false
  validKeys: ["user_preferences", "conversation_context", "personal_info"]
  tokenLimit: 2000
  charLimit: 10000
  personalize: true
  messageWindowSize: 5
  agent:
    provider: "openAI"
    model: "gpt-4"
    instructions: "您是一位得力的助手，負責記住使用者的偏好與對話上下文。"
    model_parameters:
      temperature: 0.7
      max_tokens: 1000
```

`agent` 欄位支援兩種不同的配置格式：

### 依 ID 指定智慧體

如果您已有預先配置好的智慧體，可以透過其 ID 進行引用：

```yaml filename="memory / agent (依 ID)"
memory:
  agent:
    id: "memory-agent-001"
```

### 自定義智慧體配置

若需更多控制，您可以定義完整的智慧體配置：

```yaml filename="memory / agent (自定義)"
memory:
  agent:
    provider: "openAI"
    model: "gpt-4"
    instructions: "您是一位記憶助手，協助維護對話上下文與使用者偏好。"
    model_parameters:
      temperature: 0.3
      max_tokens: 1500
      top_p: 0.9
```

## 完整配置範例

以下是顯示所有記憶配置選項的完整範例：

```yaml filename="librechat.yaml"
version: 1.3.4
cache: true

memory:
  disabled: false
  validKeys:
    - "user_preferences"
    - "conversation_context" 
    - "learned_facts"
    - "personal_information"
  tokenLimit: 3000
  charLimit: 10000
  personalize: true
  messageWindowSize: 8
  agent:
    provider: "openAI"
    model: "gpt-4"
    instructions: |
      僅使用指定的有效鍵 (validKeys) 儲存記憶。針對 user_preferences：儲存
      明確說明的關於通訊風格、感興趣的主題或工作流偏好。針對 conversation_context：
      儲存提及的重要事實或進行中的專案。針對 learned_facts：儲存關於使用者的
      客觀資訊。針對 personal_information：僅儲存使用者明確分享的關於其個人的資訊。
      請及時刪除過期或錯誤的資訊。
    model_parameters:
      temperature: 0.2
      max_tokens: 2000
      top_p: 0.8
      frequency_penalty: 0.1
```

## 使用自定義終端

記憶功能支援自定義終端。使用自定義終端時，`provider` 欄位應與自定義終端的 `name` 完全一致。包含環境變數與使用者預留位置的自定義標頭將被正確解析。

```yaml filename="包含記憶用自定義終端的 librechat.yaml"

endpoints:
    custom:
        - name: 'Custom Memory Endpoint'
           apiKey: 'dummy'
           baseURL: 'https://api.gateway.ai/v1'
           headers:
             x-gateway-api-key: '${GATEWAY_API_KEY}'
             x-gateway-virtual-key: '${GATEWAY_OPENAI_VIRTUAL_KEY}'
             X-User-Identifier: '{{LIBRECHAT_USER_EMAIL}}'
             X-Application-Identifier: 'LibreChat - Test'
             api-key: '${TEST_CUSTOM_API_KEY}' 
           models:
             default:
               - 'gpt-4o-mini'
               - 'gpt-4o'
             fetch: false
  
memory:
  disabled: false
  tokenLimit: 3000
  personalize: true
  messageWindowSize: 10
  agent:
    provider: 'Custom Memory Endpoint'
    model: 'gpt-4o-mini'
```

- 關於所有可用的預留位置，請參閱[自定義終端標頭](/docs/configuration/librechat_yaml/object_structure/custom_endpoint_zh_TW#headers)。

## 備註

- 記憶功能可增強對話連貫性與個人化體驗。
- 當 `personalize` 為 true 時，使用者可在其聊天介面中看到一個切換開關來控制記憶的使用。
- 代幣限制有助於控制記憶使用量與處理成本。
- 有效鍵提供對哪些資訊可被儲存的細粒度控制。
- 自定義 `instructions` 取代了預設的記憶處理指令，應與 `validKeys` 配合使用。
- 智慧體配置允許自定義記憶處理行為。
- 設定為停用後，所有記憶功能都將關閉，不論其他設定為何。
- 訊息窗口大小會影響在更新記憶時考量的近期上下文份量。
