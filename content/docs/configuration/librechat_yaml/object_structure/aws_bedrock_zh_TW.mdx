將 AWS Bedrock 整合至您的應用程式，讓您可以無縫利用 AWS 上託管的多種 AI 模型。本章節詳細說明如何根據您的需求配置 AWS Bedrock 終端。

## 配置範例

```yaml filename="AWS Bedrock 物件結構範例"
endpoints:
  bedrock:
    titleModel: "anthropic.claude-3-haiku-20240307-v1:0"
    streamRate: 35
    availableRegions:
      - "us-east-1"
      - "us-west-2"
    guardrailConfig:
      guardrailIdentifier: "您的-guardrail-id"
      guardrailVersion: "1"
      trace: "enabled"
```

> **備註：** AWS Bedrock 終端支援所有[共用終端設定](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings_zh_TW)，包括 `streamRate`、`titleModel`、`titleMethod`、`titlePrompt`、`titlePromptTemplate` 與 `titleEndpoint`。下方顯示的設定是 Bedrock 特有的，或具有 Bedrock 特有的預設值。

## inferenceProfiles

**Key：**
<OptionTable
  options={[
    ['inferenceProfiles', '物件 (Record)', '將模型 ID 映射至推理設定檔 (inference profile) ARN 以進行跨區域推理。鍵 (keys) 為模型 ID，值為推理設定檔 ARN 或環境變數引用。', '當選擇的模型與鍵匹配時，對應的 ARN 將被用作應用程式推理設定檔。'],
  ]}
/>

**預設值：** 未指定

**範例：**
```yaml filename="inferenceProfiles"
endpoints:
  bedrock:
    inferenceProfiles:
      "us.anthropic.claude-sonnet-4-20250514-v1:0": "${BEDROCK_INFERENCE_PROFILE_CLAUDE_SONNET}"
      "anthropic.claude-3-7-sonnet-20250219-v1:0": "arn:aws:bedrock:us-west-2:123456789012:application-inference-profile/abc123"
```

**備註：**
- 推理設定檔支援跨區域推理，允許您將請求路由至不同 AWS 區域的模型。
- 數值支援使用 `${ENV_VAR}` 語法的環境變數插值。
- 鍵中的模型 ID 必須與使用者在 UI 中選擇的模型一致。
- 搭配 `models` 欄位使用，使跨區域模型 ID 出現在模型選擇器中。
- 如需建立與管理推理設定檔的完整指南，請參閱 [AWS Bedrock 推理設定檔 (Inference Profiles)](/docs/configuration/pre_configured_ai/bedrock_inference_profiles_zh_TW)。

**綜合範例：**
```yaml filename="具備推理設定檔的 Bedrock"
endpoints:
  bedrock:
    models:
      - "us.anthropic.claude-sonnet-4-20250514-v1:0"
      - "us.anthropic.claude-haiku-4-20250514-v1:0"
    inferenceProfiles:
      "us.anthropic.claude-sonnet-4-20250514-v1:0": "${BEDROCK_CLAUDE_SONNET_PROFILE}"
      "us.anthropic.claude-haiku-4-20250514-v1:0": "${BEDROCK_CLAUDE_HAIKU_PROFILE}"
```

  ]}

**子鍵 (Sub-keys)：**
<OptionTable
  options={[
    ['guardrailIdentifier', '字串', '要套用的護欄唯一識別碼。', '使用護欄時為必填。'],
    ['guardrailVersion', '字串', '要使用的護欄版本。', '使用護欄時為必填。'],
    ['trace', '字串', '控制護欄追蹤輸出以進行偵錯。選項："enabled"、"enabled_full" 或 "disabled"。', '選填。預設值："disabled"'],
  ]}
/>

**範例：**
```yaml filename="guardrailConfig"
endpoints:
  bedrock:
    guardrailConfig:
      guardrailIdentifier: "abc123xyz"
      guardrailVersion: "1"
      trace: "enabled"
```

**備註：**
- 護欄有助於確保負責任的 AI 使用，透過過濾有害內容、PII（個人識別資訊）及其他敏感資訊。
- `guardrailIdentifier` 可在 AWS Bedrock 主控台的 Guardrails 下找到。
- 在開發期間將 `trace` 設定為 `"enabled"` 或 `"enabled_full"`，以查看觸發了哪些護欄原則。
- 在生產環境中，將 `trace` 設定為 `"disabled"` 以減少回應負載大小。

## 備註

- AWS Bedrock 的主要配置是透過環境變數完成的，額外的身份驗證方式正在開發中。
