## **概覽**

`modelSpecs` 物件協助您為應用程式內的 AI 模型提供更簡約的 UI 體驗。

`modelSpecs` 下包含 4 個主要欄位：

  - `enforce` (選填；預設值：false)
  - `prioritize` (選填；預設值：true)
  - `list` (必填)
  - `addedEndpoints` (選填)

**備註：**

- 若將 `enforce` 設定為 true，模型規範可能會與 `modelSelect`、`presets` 與 `parameters` 等介面設定發生衝突。
- `list` 陣列包含每個模型的詳細配置，包括決定特定行為、外觀與能力的預設設定 (presets)。
- 若配置中未指定 [interface](interface_zh_TW.mdx) 欄位，擁有模型規範清單將會停用下列介面元素：
    - `modelSelect` (模型選擇)
    - `parameters` (參數設定)
    - `presets` (預設設定)
- 如果您希望在啟用模型規範的同時保留這些介面元素，可以在 `interface` 物件中將它們設定為 `true`。

## 範例

```yaml filename="modelSpecs"
modelSpecs:
  enforce: true
  prioritize: true
  list:
    - name: "meeting-notes-gpt4"
      label: "會議紀錄助手 (GPT4)"
      default: true
      description: "只需貼上 Teams 錄音的逐字稿，即可產生會議紀錄。"
      iconURL: "https://example.com/icon.png"
      preset:
        endpoint: "azureOpenAI"
        model: "gpt-4-turbo-1106-preview"
        maxContextTokens: 128000 # 最大上下文代幣
        max_tokens: 4096 # 最大輸出代幣
        temperature: 0.2
        modelLabel: "會議摘要器"
        greeting: |
          此助手會根據 Teams 錄音的逐字稿建立會議紀錄。
          要開始使用，只需將逐字稿貼入聊天框即可。
        promptPrefix: |
          請根據逐字稿為商務會議建立連貫的會議紀錄。請包含下列章節：
          - 日期與與會者
          - 議程
          - 會議過程
          - 待辦事項

          重點放在討論及/或解決了哪些項目。列出所有開啟的待辦事項。
          格式應為按時間順序排列的高層級主題列點清單，然後是一句或多則精簡的句子來說明詳情。
          每個高層級主題應至少列出兩個子主題，但可根據需要增加子主題以支持高層級主題。

          - 每個項目的開頭請勿使用相同的詞彙。

          請深呼吸並確保按部就班地思考。
```

### group (群組)

<OptionTable
  options={[
    ['group', '字串', '選填的群組名稱，用於在 UI 選擇器中組織模型規範。控制規範在選單階層中的位置。', '無預設值。選填。'],
    ['groupIcon', '字串', '選填的自定義群組圖標。可以是 URL 或內建終端鍵（例如 "openAI"、"groq"）。僅每個群組中第一個帶有 groupIcon 的規範會被使用。', '無預設值。選填。'],
  ]}
/>

**說明：**
選填的群組名稱，用於在 UI 選擇器中組織模型規範。`group` 欄位提供了如何組織模型規範的靈活性：

- **若 `group` 匹配終端名稱**（例如：`"openAI"`、`"groq"`）：模型規範將嵌套在選擇器選單的該終端下方。
- **若 `group` 為自定義名稱**（不匹配任何終端）：會建立一個以此名稱命名的獨立可摺疊區段。您可以選擇使用 `groupIcon` 為該區段設定自定義圖標（URL 或內建鍵如 `"openAI"`）。
- **若省略 `group`**：模型規範將作為獨立項目出現在頂層。

當您想為模型加入描述而不希望失去選擇器選單的組織結構時，此功能特別有用。

**範例：**

```yaml filename="包含 group 欄位的 modelSpecs 範例"
modelSpecs:
  list:
    # 範例 1：嵌套在終端下方
    # 當 group 匹配終端名稱時，規範會出現在該終端下方
    - name: "gpt-4o-optimized"
      label: "GPT-4 優化版"
      description: "最具能力的 GPT-4 模型，支援多模態"
      group: "openAI"  # 出現在 OpenAI 終端下方
      preset:
        endpoint: "openAI"
        model: "gpt-4o"

    # 範例 2：具備圖標的自定義群組區段
    # 當 group 為自定義名稱時，會建立一個獨立的可摺疊區段
    - name: "coding-assistant"
      label: "程式碼助手"
      description: "專精於程式撰寫任務"
      group: "我的助手清單"
      groupIcon: "https://example.com/icons/assistants.png"  # 群組的自定義圖標
      preset:
        endpoint: "openAI"
        model: "gpt-4o"

    # 具有相同群組名稱的多個規範會被歸類在一起
    - name: "writing-assistant"
      label: "寫作助手"
      description: "專精於創意寫作"
      group: "我的助手清單"  # 與 coding-assistant 歸為同組，使用其圖標
      preset:
        endpoint: "anthropic"
        model: "claude-sonnet-4"

    # 範例 3：使用內建圖標的自定義群組
    - name: "fast-model"
      label: "快速模型"
      group: "極速模型區"
      groupIcon: "groq"  # 使用內建的 Groq 圖標
      preset:
        endpoint: "groq"
        model: "llama3-8b-8192"

    # 範例 4：獨立顯示（不分組）
    # 若省略 group 欄位，規範將出現在頂層
    - name: "general-assistant"
      label: "通用助手"
      description: "一般用途助手"
      # 無 group 欄位 - 作為獨立項目出現在頂層
      preset:
        endpoint: "openAI"
        model: "gpt-4o-mini"
```

## 預設設定欄位 (Preset Fields)

`modelSpecs.list` 項目的 `preset` 欄位由系統內 AI 模型的完整配置藍圖組成。它旨在指定 AI 模型的運作設定，量身打造其行為、輸出以及與其他系統組件和終端的互動。

### 系統選項

#### endpoint (終端)

**必填**

**接受的值：**
- `openAI`
- `azureOpenAI`
- `google`
- `anthropic`
- `assistants`
- `azureAssistants`
- `bedrock`
- `agents`

**備註：** 若使用自定義終端，`endpoint` 數值必須與定義的[自定義終端名稱](/docs/configuration/librechat_yaml/object_structure/custom_endpoint_zh_TW#name)完全一致。

  ]}
/>

**範例：**

如需更多關於 ACL 權限系統的資訊，請參閱 [智慧體文檔](/docs/features/agents_zh_TW#migration-required-v080-rc3)。
