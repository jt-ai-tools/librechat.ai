## 概覽

`fileConfig` 物件讓您可以配置應用程式的檔案處理設定，包括大小限制與 MIME 類型限制。本章節提供 `fileConfig` 物件結構的詳細解析。

`fileConfig` 下包含 8 個主要欄位：

  - `endpoints` (終端)
  - `serverFileSizeLimit` (伺服器檔案大小限制)
  - `avatarSizeLimit` (頭像大小限制)
  - `imageGeneration` (圖片產生)
  - `fileTokenLimit` (檔案代幣限制)
  - `ocr` (光學字元辨識)
  - `text` (文字解析)
  - `stt` (語音轉文字)

**備註：**

- 截至撰寫本文時，助手 (Assistants) 終端[支援此清單中的檔案類型](https://platform.openai.com/docs/assistants/tools/file-search#supported-files)。
- OpenAI、Azure OpenAI、Google 與自定義終端透過 [RAG API](../../rag_api_zh_TW) 支援檔案。
- `ocr`、`text` 與 `stt` 區段控制了針對[以文字上傳](/docs/features/upload_as_text_zh_TW)與 [OCR](/docs/features/ocr_zh_TW) 等功能的檔案處理方式。
- 任何未提及的其他終端（如外掛程式 Plugins）目前尚不支援檔案上傳。
- 助手終端的定義值為 `assistants`。所有其他終端使用定義值 `default`。
  - 對於非助手終端，您可以在 `default` 下調整所有終端的檔案設定。
  - 如果您想針對特定終端調整設定，可以列出其對應的終端名稱：
    - `assistants`
        - 不使用 "default"，因為它有獨立於其他的預設值。
    - `openAI`
    - `azureOpenAI`
    - `google`
    - `您的自定義終端名稱`
- 您可以省略數值，在此情況下，應用程式將使用下方按終端類型定義的預設值。
- LibreChat 計算 1 MB 的方式為：`1 x 1024 x 1024`。

## 範例

```yaml filename="fileConfig"
fileConfig:
  endpoints:
    assistants:
      fileLimit: 5
      fileSizeLimit: 10
      totalSizeLimit: 50
      supportedMimeTypes:
        - "image/.*"
        - "application/pdf"
    openAI:
      disabled: true
    default:
      totalSizeLimit: 20
    您的自定義終端名稱:
      fileLimit: 5
      fileSizeLimit: 1000
      supportedMimeTypes:
        - "image/.*"
  serverFileSizeLimit: 1000
  avatarSizeLimit: 2
  fileTokenLimit: 100000
  imageGeneration:
    percentage: 100
    px: 1024
  ocr:
    supportedMimeTypes:
      - "^image/(jpeg|gif|png|webp|heic|heif)$"
      - "^application/pdf$"
      - "^application/vnd\.openxmlformats-officedocument\.(wordprocessingml\.document|presentationml\.presentation|spreadsheetml\.sheet)$"
      - "^application/vnd\.ms-(word|powerpoint|excel)$"
      - "^application/epub\+zip$"
  text:
    supportedMimeTypes:
      - "^text/(plain|markdown|csv|json|xml|html|css|javascript|typescript|x-python|x-java|x-csharp|x-php|x-ruby|x-go|x-rust|x-kotlin|x-swift|x-scala|x-perl|x-lua|x-shell|x-sql|x-yaml|x-toml)$"
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

## imageGeneration (圖片產生)

<OptionTable
  options={[
    ['imageGeneration', '物件', '與圖片產生輸出品質與尺寸相關的設定。', '允許將輸出大小配置為相對於某個基礎大小的百分比，或配置為明確的像素尺寸。'],
  ]}
/>

`imageGeneration` 支援下列參數：

- `percentage` (整數)
  - 以百分比表示的所產生圖片輸出大小（例如：`100` 代表基礎大小的 100%）。
  - 用於相對於預設或原始大小縮放輸出圖片。

- `px` (整數)
  - 指定以像素為單位的輸出圖片尺寸（例如：`1024`）。
  - 用於明確設定所產生圖片的輸出大小，而不論其基礎大小為何。

您可以根據使用情境設定這兩個參數中的其中一個（`percentage` 或 `px`），不可兩者同時設定。

配置範例：

```yaml filename="fileConfig / imageGeneration"
fileConfig:
  imageGeneration:
    percentage: 100
    px: 1024
```

### supportedMimeTypes

<OptionTable
  options={[
    ['supportedMimeTypes', '字串陣列', '應使用 STT 轉錄的音訊檔案 MIME 類型模式清單。', '使用正則表達式來匹配音訊檔案類型。'],
  ]}
/>

**預設值：** 常見音訊格式。

```yaml filename="fileConfig / stt / supportedMimeTypes"
fileConfig:
  stt:
    supportedMimeTypes:
      - "^audio/(mp3|mpeg|mpeg3|wav|wave|x-wav|ogg|vorbis|mp4|x-m4a|flac|x-flac|webm)$"
```

**備註：**
- 匹配 `text` 模式的檔案會以簡單文字提取進行處理。
- 匹配 `ocr` 模式的檔案會使用提供的 OCR 服務進行處理。
- 匹配 `stt` 模式的檔案會使用語音轉文字轉錄進行處理。
- **處理優先權：OCR > STT > 文字解析 > 備援 (fallback)**
- 未匹配任何模式的檔案將退而使用文字解析。

## 檔案處理優先順序

LibreChat 根據 MIME 類型匹配處理上傳的檔案，**優先順序**如下：

1. **OCR** - 若檔案匹配 `ocr.supportedMimeTypes` 且已配置 OCR。
2. **STT** - 若檔案匹配 `stt.supportedMimeTypes` 且已配置 STT。
3. **文字解析 (Text Parsing)** - 若檔案匹配 `text.supportedMimeTypes`。
4. **備援 (Fallback)** - 作為最後手段的文字解析。

此處理順序確保了在未配置專門服務 (OCR/STT) 的情況下仍能維持功能，同時在可用時提供最佳的提取品質。

### 處理範例

**配置了 OCR 的 PDF 檔案：**
- 檔案匹配 `ocr.supportedMimeTypes`。
- **使用 OCR** 提取文字。
- 對於掃描的 PDF 與圖片有更好的品質。

**未配置 OCR 的 PDF 檔案：**
- 檔案匹配 `text.supportedMimeTypes`（或使用備援）。
- **使用文字解析** 函式庫。
- 對於具備可選取文字的數位 PDF 運作良好。

**Python 檔案：**
- 檔案匹配 `text.supportedMimeTypes`。
- **使用文字解析**（無需 OCR）。
- 直接提取文字。

**配置了 STT 的音訊檔案：**
- 檔案匹配 `stt.supportedMimeTypes`。
- **使用 STT** 將音訊轉錄為文字。

**未配置 OCR 的圖片檔案：**
- 檔案匹配 `ocr.supportedMimeTypes` 但 OCR 不可用。
- **退而使用文字解析**。
- 無 OCR 時提取能力有限。

此優先系統允許「以文字上傳」等功能在無需 OCR 配置的情況下運作，同時在 OCR 可用時利用其提升提取品質。

## endpoints (終端)

<OptionTable
  options={[
    ['endpoints', '紀錄/物件', '為個別終端配置檔案處理設定，允許按終端進行自定義。', '指定個別終端的檔案處理配置。'],
  ]}
/>

**說明：** endpoints 下的每個物件都是一條紀錄，可包含下列設定：

### 概覽

  - `disabled`
      - 是否針對該終端停用檔案處理。
  - `fileLimit`
      - 每次上傳請求允許的最大檔案數量。
  - `fileSizeLimit`
      - 單個檔案的最大大小。以 MB 為單位（例如：使用 `20` 代表 20 MB）。
  - `totalSizeLimit`
      - 單次請求中所有檔案的最大總大小。以 MB 為單位。
  - `supportedMimeTypes`
      - 指定允許上傳之 MIME 類型的[正則表達式 (Regular Expressions)](https://zh.wikipedia.org/wiki/%E6%AD%A3%E5%88%99%E8%A1%A8%E8%BE%BE%E5%BC%8F)清單。可用於限制檔案類型。
