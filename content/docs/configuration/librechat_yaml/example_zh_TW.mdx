## 帶有註釋的範例

此範例配置檔案在幾個關鍵領域為 LibreChat 進行了詳細設定：

- **快取 (Caching)**：已啟用以提升性能。
- **檔案處理 (File Handling)**：
    - **檔案策略 (File Strategy)**：已註釋，但提示了可能與 Firebase 整合進行檔案儲存。
    - **檔案配置 (File Configurations)**：針對不同終端自定義了檔案上傳限制與允許的 MIME 類型，包括全域伺服器檔案大小限制以及使用者頭像圖片的特定限制。
- **速率限制 (Rate Limiting)**：定義了每個 IP 與使用者在指定時間窗口內允許的最大檔案上傳次數，旨在防止濫用。
- **註冊 (Registration)**：
    - 允許來自特定社群登入供應商與電子郵件網域的註冊，增強了安全性與使用者管理。
- **終端 (Endpoints)**：
    - **Assistants**：配置了 Assistants 終端的輪詢間隔與操作超時，並提供了停用構建器介面的選項。
    - **自定義終端 (Custom Endpoints)**：
        - 配置了兩個外部 AI 服務終端：Mistral 與 OpenRouter，包括 API 金鑰、基礎 URL、模型處理以及特定的功能切換（如對話標題、摘要與參數調整）。
        - 針對 Mistral，啟用了動態模型獲取，套用了安全提示詞的額外參數，並明確刪除了不支援的參數。
        - 針對 OpenRouter，設定了不含動態模型獲取的基礎配置，並指定了用於對話標題的模型。
