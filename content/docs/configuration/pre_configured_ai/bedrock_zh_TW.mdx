請前往 [AWS 官方文檔](https://docs.aws.amazon.com/bedrock/latest/userguide/getting-started.html) 註冊 AWS 並設定您的憑證。

您還需要為您的帳號開啟模型存取權限，您可以[遵循此處說明](https://docs.aws.amazon.com/bedrock/latest/userguide/model-access.html)進行操作。

## 身份驗證

- 您需要設定下列環境變數：

```bash filename=".env"
BEDROCK_AWS_DEFAULT_REGION=us-east-1
BEDROCK_AWS_ACCESS_KEY_ID=您的_access_key_id
BEDROCK_AWS_SECRET_ACCESS_KEY=您的_secret_access_key
```

**備註：** 您也可以省略存取金鑰以使用預設的 AWS 憑證鏈，但必須設定預設區域：

```bash filename=".env"
BEDROCK_AWS_DEFAULT_REGION=us-east-1
```

執行此操作會提示憑證供應商按下列順序（優先級由高到低）尋找憑證：

- 透過 process.env 公開的環境變數
- 來自權杖快取的 SSO 憑證
- 網頁識別權杖 (Web identity token) 憑證
- 共用憑證與配置的 ini 檔案
- EC2/ECS 實例元數據服務 (Instance Metadata Service)

預設憑證供應商會依序調用供應商，僅在未找到憑證時才會繼續嘗試下一個。

例如，若程序找到了透過 `AWS_ACCESS_KEY_ID` 與 `AWS_SECRET_ACCESS_KEY` 環境變數定義的數值，則不會讀取 `~/.aws/credentials` 與 `~/.aws/config` 檔案，也不會向實例元數據服務發送任何訊息。

## 配置模型

- 您可以選擇性地透過 `BEDROCK_AWS_MODELS` 指定想提供的模型：

```bash filename=".env"
BEDROCK_AWS_MODELS=anthropic.claude-3-5-sonnet-20240620-v1:0,meta.llama3-1-8b-instruct-v1:0
```

備註：若省略此項，將自動包含所有已知的支援模型 ID。

- 在此處查看所有 Bedrock 模型 ID：
    - **[https://docs.aws.amazon.com/bedrock/latest/userguide/model-ids.html#model-ids-arns](https://docs.aws.amazon.com/bedrock/latest/userguide/model-ids.html#model-ids-arns)**

## 額外配置

您可以在 [`librechat.yaml`](/docs/configuration/librechat_yaml_zh_TW) 檔案中進一步配置 Bedrock 終端：

```yaml
endpoints:
  bedrock:
    availableRegions:
      - "us-east-1"
      - "us-west-2"
    streamRate: 35
    titleModel: 'anthropic.claude-3-haiku-20240307-v1:0'
    guardrailConfig:
      guardrailIdentifier: "abc123xyz"
      guardrailVersion: "1"
      trace: "enabled"
```

- `streamRate`：（選填）設定處理每個新權杖 (token) 的速率（毫秒）。
    - 這能協助穩定並行請求的處理，並提供更流暢的前端串流渲染。

- `titleModel`：（選填）指定用於產生對話標題的模型。
    - 推薦：`anthropic.claude-3-haiku-20240307-v1:0`。
    - 若省略或設定為 `current_model`，則使用與聊天相同的模型。

- `availableRegions`：（選填）指定您想提供的 AWS 區域。
    - 若有提供，使用者將在下拉選單中看到區域選擇。若未選擇，則使用預設區域。
    - ![image](https://github.com/user-attachments/assets/6f3c5e82-9c6b-4643-8487-07db1061ba49)

- `guardrailConfig`：（選填）配置 AWS Bedrock 護欄 (Guardrails) 以進行內容過濾。
    - `guardrailIdentifier`：來自您的 AWS Bedrock 主控台的護欄 ID 或 ARN。
    - `guardrailVersion`：護欄版本號（例如：`"1"`）或 `"DRAFT"`。
    - `trace`：（選填）啟用追蹤日誌：`"enabled"`、`"disabled"` 或 `"enabled_full"`。
    - 關於建立與管理護欄，請參閱 [AWS Bedrock 護欄文檔](https://docs.aws.amazon.com/bedrock/latest/userguide/guardrails-how.html)。

## 推理設定檔 (Inference Profiles)

AWS Bedrock 推理設定檔讓您可以為基礎模型建立自定義路由配置，實現跨區域負載平衡、成本分配與合規控制。您可以在 `librechat.yaml` 中將模型 ID 映射至自定義的推理設定檔 ARN：

```yaml
endpoints:
  bedrock:
    inferenceProfiles:
      "us.anthropic.claude-3-7-sonnet-20250219-v1:0": "${BEDROCK_CLAUDE_37_PROFILE}"
```

關於建立設定檔、配置 LibreChat、設定日誌紀錄與疑難排解的完整指南，請參閱 **[Bedrock 推理設定檔](/docs/configuration/pre_configured_ai/bedrock_inference_profiles_zh_TW)**。

關於 YAML 欄位參考，請參閱 **[AWS Bedrock 物件結構](/docs/configuration/librechat_yaml/object_structure/aws_bedrock_zh_TW#inferenceprofiles)**。

## 備註

- 下列模型因不具備串流能力而不受支援：
  - ai21.j2-mid-v1

- 下列模型因不支援對話歷史而不受支援：
  - ai21.j2-ultra-v1
  - cohere.command-text-v14
  - cohere.command-light-text-v14

- AWS Bedrock 終端支援透過 `librechat.yaml` 配置文件設定所有[共用終端設定](/docs/configuration/librechat_yaml/object_structure/shared_endpoint_settings_zh_TW)，包含 `streamRate`、`titleModel`、`titleMethod`、`titlePrompt`、`titlePromptTemplate` 與 `titleEndpoint`。
