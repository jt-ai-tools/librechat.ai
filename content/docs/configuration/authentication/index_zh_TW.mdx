## 自動審核系統 (選填)

自動審核系統預設為啟用。它使用評分機制來追蹤使用者違規行為。當使用者執行如過度登入、註冊或傳送訊息等操作時，會累積違規分數。一旦達到設定的閾值，該使用者及其相關 IP 將被暫時封鎖。此系統透過監測並懲罰快速或可疑的活動來確保平台安全。

要設定審核系統，請參閱 [設定指南](/docs/configuration/mod_system_zh_TW)。

> *請注意：若您想讓此功能在開發模式下運作，您需要在根目錄建立一個名為 `.env.development` 的檔案，並將 `DOMAIN_CLIENT` 設定為 `http://localhost:3090`，或設定為執行 `npm run frontend-dev` 時由 vite 提供的任何連接埠。*

## 使用者管理腳本

### 建立使用者腳本

建立使用者 (create-user) 腳本讓您可以直接將使用者加入資料庫，即使已停用註冊功能。使用方法如下：

1. 使用預設的 `docker-compose.yml` 時（若您使用 `docker compose up` 啟動應用程式）：
   ```bash
   docker-compose exec api npm run create-user
   ```

2. 使用 `deploy-compose.yml` 時（若您參考 [Ubuntu Docker 指南](/docs/remote/docker_linux_zh_TW)）：
   ```bash
   docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run create-user"
   ```

3. 用於本地開發時（在專案根目錄執行）：
   ```bash
   npm run create-user
   ```

按照提示輸入新使用者的電子郵件與密碼。

### 刪除使用者腳本

若要刪除使用者，您可以使用刪除使用者 (delete-user) 腳本：

1. 使用預設的 `docker-compose.yml` 時（若您使用 `docker compose up` 啟動應用程式）：
   ```bash
   docker-compose exec api npm run delete-user email@domain.com
   ```

2. 使用 `deploy-compose.yml` 時（若您參考 [Ubuntu Docker 指南](/docs/remote/docker_linux_zh_TW)）：
   ```bash
   docker exec -it LibreChat-API /bin/sh -c "cd .. && npm run delete-user email@domain.com"
   ```

3. 用於本地開發時（在專案根目錄執行）：
   ```bash
   npm run delete-user email@domain.com
   ```

將 `email@domain.com` 替換為您想要刪除的使用者電子郵件。
