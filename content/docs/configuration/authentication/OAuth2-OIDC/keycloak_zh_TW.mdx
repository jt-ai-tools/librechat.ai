title: Keycloak
icon: KeyRound
description: 學習如何配置 LibreChat 使用 Keycloak 進行使用者身份驗證。
---

1. **存取 Keycloak 管理主控台：**
- 在網頁瀏覽器中開啟 Keycloak 管理主控台。通常位於如下 URL：`http://localhost:8080/auth/admin/`。

2. **建立領域 (Realm)（如有必要）：**
- 如果您尚未為應用程式建立領域，請先建立一個。點擊 「Add Realm」 並給予一個名稱。

3. **建立用戶端 (Client)：**
- 在您的領域中，點擊 「Clients」，接著點擊 「Create」。
- 輸入用戶端 ID (Client ID)，並選擇 「openid-connect」 作為用戶端協定 (Client Protocol)。
- 將 「Client Authentication」 設定為 「On」。
- 在 「Valid Redirect URIs」 中，輸入 `http://localhost:3080/oauth/openid/callback` 或適合您應用程式的 URI。

4. **配置用戶端：**
- 建立用戶端後，您將被重新導向至其設定頁面。
- 記下 「Credentials」 標籤中的 「Client ID」 與 「Secret」 —— 您在應用程式中會需要這些資訊。

![image](https://github.com/danny-avila/LibreChat/assets/6623884/b1c1f0b6-641b-4cf7-a7f1-a9a32026d51b)


5. **新增角色（選填）：**
如果您想將存取權限限制給具備特定角色的使用者，您可以在 Keycloak 中定義角色並指派給使用者。
- 前往用戶端或領域中的 「Roles」 標籤（取決於您想在哪裡定義角色）。
- 建立與您在 `OPENID_REQUIRED_ROLE` 中設定的數值相符的角色。

![image](https://github.com/danny-avila/LibreChat/assets/6623884/67ca635f-5082-4dcc-97ac-019029a81d7c)

6. **將角色指派給使用者（選填）：**
- 前往 「Users」，選擇一名使用者，並前往 「Role Mappings」 標籤。
- 為使用者指派至少一個在 `OPENID_REQUIRED_ROLE` 中指定的角色。

![image](https://github.com/danny-avila/LibreChat/assets/6623884/f2ea70ed-e16c-4ec8-b84f-79fbfca627be)

7. **獲取權杖內的角色列表路徑（選填）：**
- 解碼來自 OpenID 供應商的 jwtToken，並確定存取權杖 (access token) 內角色列表的路徑。例如，如果您使用的是 Keycloak，該路徑為 `realm_access.roles`。
- 將此路徑填入 `.env` 檔案中的 `OPENID_REQUIRED_ROLE_PARAMETER_PATH` 變數。
- 透過參數 `OPENID_REQUIRED_ROLE_TOKEN_KIND`，您可以指定要使用的權杖種類。可能的值為 `access` 與 `id`。

8. **更新您的專案配置：**
- 開啟專案資料夾中的 `.env` 檔案，並新增下列變數：
  ```bash filename=".env"
  OPENID_ISSUER=http://localhost:8080/realms/[您的領域名稱]
  OPENID_CLIENT_ID=[您的ClientID]
  OPENID_CLIENT_SECRET=[您的ClientSecret]
  OPENID_SESSION_SECRET=[請產生一個隨機工作階段秘密]
  OPENID_CALLBACK_URL=/oauth/openid/callback
  OPENID_SCOPE="openid profile email"
  OPENID_REQUIRED_ROLE=[您要求的角色名稱] # 單一角色或多個以逗號分隔的角色 (例如：role1,role2,admin)
  OPENID_REQUIRED_ROLE_TOKEN_KIND=(access|id) # 代表 `access` 或 `id`
  OPENID_REQUIRED_ROLE_PARAMETER_PATH="realm_access.roles"

  # 選填：登出後將使用者重新導向至結束工作階段終端
  OPENID_USE_END_SESSION_ENDPOINT=true 
  ```
