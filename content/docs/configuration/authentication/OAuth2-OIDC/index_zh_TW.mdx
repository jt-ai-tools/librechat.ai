## OpenID Connect 疑難排解

如果您在 OpenID Connect 身份驗證中遇到問題：

1. **啟用標頭偵錯日誌**：在環境變數中設定 `DEBUG_OPENID_REQUESTS=true`，除了請求 URL 外，還會紀錄請求標頭（敏感資料會被遮罩）。備註：請求 URL 始終會在偵錯 (debug) 層級紀錄。
2. **檢查重新導向 URI**：確保您的回呼 (callback) URL 在供應商設定與 LibreChat 配置中完全一致。
3. **驗證範圍 (Scopes)**：確保所有必要的範圍均已正確配置。
4. **檢閱供應商日誌**：檢查您的身份供應商日誌以獲取身份驗證錯誤資訊。
5. **驗證權杖 (Tokens)**：確保您的供應商核發的權杖有效且包含預期的聲明 (claims)。
6. **確保產生 _nonce_**：部分身份供應商若在請求中缺少 `nonce` URL 參數會自行產生。設定 `OPENID_GENERATE_NONCE=true` 以強制 openid-client 產生該參數。
