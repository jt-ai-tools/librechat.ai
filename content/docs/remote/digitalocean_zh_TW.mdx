## 第一部分：從零開始：

### **1. 註冊 DigitalOcean**

[點擊此處](https://m.do.co/c/4486923fcf00) 或上方橫幅即可開始。

登入後，您會看到一個 [精美的歡迎畫面](https://cloud.digitalocean.com/welcome)。

![image](https://github.com/danny-avila/LibreChat/assets/110412045/b7a71eae-770e-4c69-a5d4-d21b939d64ed)

### **a) 導覽至專案 (Projects) 頁面**

點擊 「Explore our control panel」 或直接前往 [專案頁面](https://cloud.digitalocean.com/projects)。

在 DigitalOcean 中，伺服器實例被稱為 **「Droplets」**，它們被組織在 **「Projects (專案)」** 之下。

### **b) 點擊 「Spin up a Droplet」 開始設定**

![image](https://github.com/danny-avila/LibreChat/assets/110412045/6046e8cd-ff59-4795-a29a-5f44ab2f0a6d)

根據您的需求調整這些設定，以下我選取的是能運作的最基本/最便宜選項：

- **選擇區域/資料中心 (Choose Region/Datacenter)**：選擇距離您與您的使用者最近的位置。
- **選擇映像 (Choose an image)**：Ubuntu 22.04 (LTS) x64。
- **選擇大小 (Choose Size)**：Shared CPU, Basic Plan。
  - CPU 選項：Regular，每月 6 美元選項（每小時 0.009 美元，1 GB RAM / 1 CPU / 25 GB SSD / 1000 GB 傳輸量）。
  - 無需額外儲存空間。
- **選擇身份驗證方式 (Choose Authentication Method)**：密碼 (Password) 選項最簡單，但隨您個人偏好。
  - 或者，您也可以設定傳統的 SSH。
- **推薦 (Recommended)**：加入改進的指標監控與警示（免費）。
  - 您或許可以透過不勾選此項來使用每月 4 美元的方案，但尚未測試。
- **最後確認 (Finalize Details)**：
  - 將主機名稱 (hostname) 更改為您喜歡的名稱，其他保持預設（1 個 Droplet，無標籤）。
  - 最後點擊 「Create Droplet」。

![image](https://github.com/danny-avila/LibreChat/assets/110412045/ac90d40e-3ac6-482f-885c-58058c5e3f76)

建立 Droplet 後，系統會顯示進度條開始啟動。

### **2. 存取您的 Droplet 主控台**

啟動完成後，**點擊該 Droplet** 並點擊右側的 「Console」 連結來開啟主控台。

![image](https://github.com/danny-avila/LibreChat/assets/110412045/47c14280-fe48-49b9-9997-ff4d9c83212c)

![image](https://github.com/danny-avila/LibreChat/assets/110412045/d5e518fd-4941-4b35-86cc-69f8f65ec8eb)

透過這種方式啟動 Droplet 主控台是最簡單的方法，但如果您在先前步驟中有設定 SSH，也可以使用 SSH。

為了保持本指南的簡潔，我們將繼續使用 Droplet 主控台。如果您對 SSH 有興趣，這裡有 [DigitalOcean 的 SSH 官方指南](https://docs.digitalocean.com/products/droplets/how-to/connect-with-ssh/)。

### **3. 登入後，立即建立一個新的非 root 使用者：**

**備註：** 在本指南中，若看到大於/小於符號，請將其移除。

```bash
# 範例：adduser danny
adduser <您的使用者名稱>
# 系統隨後會提示您輸入密碼與使用者詳細資訊
```

完成後，執行下列指令提升該使用者的權限：

```bash
# 範例：usermod -aG sudo danny
usermod -aG sudo <您的使用者名稱>
```

**再次檢查您是否具備 sudo 權限，確保操作正確：**

```bash
getent group sudo | cut -d: -f4
```

**切換至新使用者**

```bash
# 範例：su - danny
su - <您的使用者名稱>
```

### **4. 防火牆設定**

再次從專案頁面點擊您的 Droplet，並前往左側 IPv4 下方的 「Networking」 標籤：
