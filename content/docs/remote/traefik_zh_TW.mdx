[Traefik](https://traefik.io/) 是一款現代化的 HTTP 反向代理與負載平衡器，讓部署與管理服務變得非常簡單。如果您在 Docker 上執行 LibreChat，可以使用 Traefik 透過 HTTPS 安全地暴露您的實例，並享有自動 SSL 憑證管理。

## 先決條件

- 您的系統已安裝 Docker 與 Docker Compose。
- 一個指向您伺服器 IP 位址的網域名稱。

## 配置

### 配置 Traefik 與 LibreChat

    在您的 `docker-compose.override.yml` 檔案中加入下列配置：

```yaml filename="docker-compose.override.yml"
version: '3'

services:
    api:
      labels:
        - "traefik.enable=true"
        - "traefik.http.routers.librechat.rule=Host(`您的.網域.名稱`)"
        - "traefik.http.routers.librechat.entrypoints=websecure"
        - "traefik.http.routers.librechat.tls.certresolver=leresolver"
        - "traefik.http.services.librechat.loadbalancer.server.port=3080"
      networks:
        - librechat_default
      volumes:
        - ./librechat.yaml:/app/librechat.yaml
  
    traefik:
      image: traefik:v3.6
      ports:
        - "80:80"
        - "443:443"
      volumes:
        - "/var/run/docker.sock:/var/run/docker.sock:ro"
        - "./letsencrypt:/letsencrypt"
      networks:
        - librechat_default
      command:
        - "--log.level=DEBUG"
        - "--api.insecure=true"
        - "--providers.docker=true"
        - "--providers.docker.exposedbydefault=false"
        - "--entrypoints.web.address=:80"
        - "--entrypoints.websecure.address=:443"
        - "--certificatesresolvers.leresolver.acme.tlschallenge=true"
        - "--certificatesresolvers.leresolver.acme.email=您的@email.com"
        - "--certificatesresolvers.leresolver.acme.storage=/letsencrypt/acme.json"

# 其他配置 #

# 備註：此區段需位於 docker-compose.override.yml 的最底部
networks:
  librechat_default:
    external: true
```

  將 `您的@email.com` 替換為您的電子郵件地址，以便接收 Let's Encrypt 的憑證通知。

  詳情請參閱：[Docker 覆寫](/docs/configuration/docker_override_zh_TW)。

### 啟動容器

  ```bash filename="啟動容器"
  docker-compose up -d
  ```

  這將啟動 Traefik 與 LibreChat 容器。Traefik 會自動從 Let's Encrypt 獲取 SSL/TLS 憑證，並透過 HTTPS 安全地暴露您的 LibreChat 實例。

您現在可以透過 `https://您的.網域.名稱` 存取您的 LibreChat 實例。Traefik 將處理 SSL/TLS 終止並將請求反向代理至您的 LibreChat 容器。

## 額外備註

- Traefik 配置分別監聽 80 與 443 連接埠以處理 HTTP 與 HTTPS 流量。請確保伺服器的防火牆已開啟這些連接埠。
- Traefik 將 SSL/TLS 憑證儲存在主機機器的 `./letsencrypt` 目錄中。建議定期備份此目錄。
- 如需更進階的配置選項，請參考 Traefik 官方文件：[https://doc.traefik.io/](https://doc.traefik.io/)

## 靜態檔案快取與壓縮

LibreChat 現已原生支援靜態檔案快取與壓縮。如果您正使用 Traefik 處理壓縮，應在 LibreChat 中停用壓縮以避免重複處理。您可以透過在 LibreChat 配置中將 `DISABLE_COMPRESSION` 環境變數設定為 `true` 來實現。

```.env
# .env 檔案
DISABLE_COMPRESSION=true
```

這將防止 LibreChat 壓縮靜態檔案，讓 Traefik 能更高效地處理壓縮工作。

關於 LibreChat 中靜態檔案處理的更多資訊（包括快取選項），請參閱 [靜態檔案處理](/docs/configuration/dotenv_zh_TW#靜態檔案處理) 文件。
