歡迎閱讀 LibreChat 部署入門指南。本文件提供了初步概覽，包含比較表以及各項詳細指南的引用，確保您對部署策略有通盤的瞭解。

在本指南中，您將探索各種選項，以便根據您的特定需求，在不同環境中高效地部署 LibreChat。

## 最低需求

執行 LibreChat 的最低需求為：

- 1 GiB RAM
- 1 vCPU

**備註：** 若啟用所有功能，建議將 RAM 增加至 2GB 以獲得更流暢的體驗。

| **服務**                           | **領域**                  | **優點**                                            | **缺點**                                         | **備註**                                    |

**Cloudflare** 以其龐大的網路聞名，能加速並保護網路服務，具備直觀的使用者介面，並在付費方案中提供強大的安全選項。

**Ngrok** 因其簡單性與能快速將本地伺服器暴露至網際網路的能力而備受讚譽，是演示與測試的理想選擇。

**Nginx** 是一款高效能網頁伺服器，資源處理效率高且穩定。然而，它需要手動設定，且相較於其他伺服器，其模組與擴充功能較少。

**Traefik** 以其自動配置更新與在容器環境中的簡易部署而聞名，因其與各種後端整合及動態重新配置的能力而吸引了 DevOps 工程師。它在微服務架構中表現出色，但對於雲端原生技術的新手來說可能具有挑戰性。

## 雲端供應商整合與配置

與雲端供應商的整合程度各不相同：從支援一鍵部署 LibreChat 的平台如 [Railway](/docs/remote/railway_zh_TW)，到利用基礎設施即代碼 (IaC) 工具（如搭配 Terraform 的 Azure）的平台，再到需要手動配置的傳統 VM 設定（如 [DigitalOcean](/docs/remote/digitalocean_zh_TW)、Linode 與 Hetzner）。

## 基本安全考量

踏入數位領域會面臨眾多威脅，危及您線上資產的安全與完整性。為了守護您的數位領地，實施強大的安全措施至關重要。

在全球規模部署應用程式時，必須考慮下列關鍵因素以確保數位資產受到保護：

1. **傳輸中數據加密**：實施具備 SSL 憑證的 HTTPS 對於防止數據被攔截與竊聽攻擊至關重要。
2. **全球存取的影響**：瞭解在全球部署應用程式的影響，包括隨區域而異的法律與合規要求。
3. **安全配置**：確保您的應用程式經過安全配置，包括使用安全協定、安全身份驗證與授權機制。

如果您選擇使用 IaaS 或隧道服務進行部署，您可能需要利用反向代理，例如 [Nginx](/docs/remote/nginx_zh_TW)、[Traefik](/docs/remote/traefik_zh_TW) 或 [Cloudflare](/docs/remote/cloudflare_zh_TW) 等。

投入適當的安全措施對於保護您的數位資產並確保全球部署的成功至關重要。

## 選擇雲端供應商（平台）

針對「正式部署」選擇雲端供應商至關重要，因為這會影響成本、效能、安全性與擴充性。您應考量資料中心位置、產業標準合規性、與現有工具的相容性以及客戶支援等因素。

市面上有許多在各方面都有所不同的選項。在本章節中，您可以找到團隊與社群使用的一些選項，這些選項能協助您進行首次部署。一旦您對應用程式的使用情況與受眾有了更多瞭解，您將能決定長期而言最適合您的雲端供應商。

如前所述，雲端供應商/平台在各方面都有所不同。就我們的目的而言，我們可以假設您的主要考量是易用性、安全性與（初始）成本。如果您還有其他疑慮（如擴充性、對特定平台的先前經驗或任何其他特定功能），那麼您可能更清楚哪種平台適合您，可以直接跳到您尋找的資訊，無需遵循特定指南。

## 根據您的需求選擇合適的部署選項

部署選項按「投入心力與掌控程度」由多到少排列：

> 每個部署選項都有其優缺點，選擇最終取決於您專案的具體需求。

### 1. IaaS (基礎設施即服務)

基礎設施即服務 (IaaS) 指的是一種雲端運算模型，按使用量付費提供基本的運算資源，如虛擬伺服器、網路與儲存空間。IaaS 允許組織透過網際網路租用並存取這些資源，而無需投資與維護實體硬體。此模型提供了擴充性、靈活性與成本節約，並能快速且輕鬆地根據不斷變化的業務需求部署與管理基礎設施資源。

- [DigitalOcean](/docs/remote/digitalocean_zh_TW)：介面友善且定價可預測。
- Linode：以卓越的客戶支援與簡單明瞭的定價聞名。

#### 針對 IaaS 我們推薦使用 Docker Compose

**為什麼選擇 Docker Compose？** 我們推薦 Docker Compose 以實現一致的部署。本指南清晰列出了每個步驟，以便於部署：[Docker - Linux 遠端安裝指南](/docs/remote/docker_linux_zh_TW)

**備註：** 儲存庫中有兩個 docker compose 檔案：

1. **開發導向的 docker compose `docker-compose.yml`**
2. **部署導向的 docker compose `deploy-compose.yml`**

主要區別在於 `deploy-compose.yml` 包含了 Nginx，使其配置成為 Docker 內部的一部分。

> 詳情請參閱 [Nginx 指南](/docs/remote/nginx_zh_TW)。

### 2. IaC (基礎設施即代碼)

基礎設施即代碼 (IaC) 指的是透過機器可讀的定義檔案來管理與置備運算基礎設施的做法，而非透過實體硬體配置或互動式配置工具。此方法促進了可重複性、可處置性與擴充性，特別是在現代雲端環境中。IaC 實現了基礎設施部署、配置與管理的自動化，從而實現更快、更一致且更可靠的資源置備。

- Azure：適合企業級部署的全面服務。

**備註：** Digital Ocean, Linode, Hetzner 也支援 IaC。雖然我們目前缺乏特定指南，但您可以嘗試調整針對 Terraform 的 Azure 指南，並協助我們貢獻改進。

### 3. PaaS (平台即服務)

平台即服務 (PaaS) 是一種雲端運算模型，在雲端提供開發與部署環境。它為開發者提供了一個建置、測試與部署應用程式的平台，而無需管理底層基礎設施。PaaS 通常包含一系列資源，如資料庫、中間件與開發工具，讓使用者能交付從簡單的雲端 App 到複雜的企業應用程式。此模型縮短了上市時間、降低了成本，並簡化了維護與擴充，因為服務供應商負責維護基礎設施，而客戶可以專注於建置、部署與管理其應用程式。

- [Hugging Face](/docs/remote/huggingface_zh_TW)：專為機器學習與 NLP 專案量身打造。
- Render：透過整合的 CI/CD 流程簡化部署。
- Heroku：適合新創公司與快速部署場景。

### 4. 一鍵部署 (PaaS)

- [Railway](/docs/remote/railway_zh_TW)：受歡迎的一鍵部署解決方案。
- Zeabur：引領潮流的簡易一鍵部署解決方案。

## 其他 / 網路服務

### 1. 隧道服務 (Tunneling)

隧道服務允許您將本地開發伺服器暴露至網際網路，使其可透過公開 URL 存取。這對於分享成果、測試以及與第三方服務整合特別有用。它讓您能部署您的開發電腦進行測試或內部安裝。

- [Ngrok](/docs/remote/ngrok_zh_TW)：協助實現通往網際網路的安全本地隧道。
- [Cloudflare](/docs/remote/cloudflare_zh_TW)：增強網頁效能與安全性。

### 2. DNS 服務

- Cloudflare DNS 服務用於管理並將網路流量導向至正確目的地，透過將人類可讀的網域名稱轉換為機器可讀的 IP 位址來實現。Cloudflare 是此服務的供應商，提供廣泛的功能，如安全性、效能與可靠性。Cloudflare DNS 服務提供了友善的介面來管理 DNS 紀錄，並提供進階功能如流量管理、DNSSEC 與 DDoS 防護。

另請參閱：[Cloudflare 指南](/docs/remote/cloudflare_zh_TW)
