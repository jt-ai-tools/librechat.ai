- 為智慧體終端新增了 `context` 能力
  - 同時啟用聊天中的「以文字上傳」功能與智慧體的「檔案上下文」功能。
  - 允許使用者上傳檔案，並將其內容提取後直接納入對話或智慧體指令中。
  - 預設使用文字解析運作，無需外部服務。
  - 當配置了 OCR 服務時，由 `ocr` 能力提供增強。
  - 在能力清單中預設為啟用。
  - 詳情請參閱 [智慧體配置](/docs/configuration/librechat_yaml/object_structure/agents_zh_TW#capabilities)。

- 在 `fileConfig` 中新增了 OCR、文字解析與 STT 的分離設定：
  - 新增了 `ocr` 配置，用以控制哪些檔案類型使用 OCR 處理。
  - 新增了 `text` 配置，用以控制哪些檔案類型使用直接文字提取。
  - 新增了 `stt` 配置，用以控制哪些音訊檔案類型使用語音轉文字轉錄。
  - 新增了 `fileTokenLimit` 參數，用以控制檔案文字提取的最大代幣限制。
  - 為視覺文件 (OCR)、文字檔案（原生解析）與音訊檔案 (STT) 提供了獨立的處理路徑。
  - 處理優先順序：OCR > STT > 文字解析 > 備援 (fallback)。
  - 預設 OCR 支援：圖片 (JPEG, GIF, PNG, WebP, HEIC, HEIF)、PDF、Office 文件、EPUB 檔案。
  - 預設文字解析支援：所有文字 MIME 類型與常見程式語言。
  - 預設 STT 支援：音訊格式 (MP3, WAV, FLAC, OGG, M4A, WebM 等)。
  - 來自附件檔案的文字會在執行階段構建提示詞前，被截斷至 `fileTokenLimit` 所設的長度（預設：100000）。
  - 詳情請參閱 [檔案配置物件結構](/docs/configuration/librechat_yaml/object_structure/file_config_zh_TW)。

- 新增了 `transactions` (交易) 物件結構
  - 控制是否將代幣使用紀錄儲存至資料庫。
  - 可獨立於餘額系統進行配置。
  - 當 `balance.enabled` 設定為 `true` 時會自動啟用。
  - 預設值為 `true`，以確保預設情況下會追蹤代幣使用量。
  - 停用交易紀錄可減少資料庫儲存空間，但將無法進行歷史使用分析。
  - 詳情請參閱 [交易配置](/docs/configuration/librechat_yaml/object_structure/transactions_zh_TW)。

- 在 `webSearch` 配置中新增了 `jinaApiUrl`
  - Jina API URL 的環境變數名稱。
  - 選填欄位，僅在您使用自定義 Jina 實例時才需要。
  - 若未在 .env 中設定，系統會提示使用者透過 UI 提供。
  - 詳情請參閱 [網頁搜尋配置](/docs/configuration/librechat_yaml/object_structure/web_search_zh_TW#jinaapiurl)。
